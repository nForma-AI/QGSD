### Task 2 — Circuit breaker lifecycle tests

Add describe block `circuit-breaker --disable-breaker / --enable-breaker` with 3 tests.

These tests call `node bin/install.js --disable-breaker` / `--enable-breaker`, NOT gsd-tools. Add constant:
```js
const INSTALL_PATH = path.join(__dirname, '../../bin/install.js');
```

Note: `--disable-breaker` uses `git rev-parse --show-toplevel` to find project root; since tmpDir is not a git repo, it falls back to `process.cwd()`. Run the command with `cwd: tmpDir` so the fallback writes to `tmpDir/.claude/`.

Use `runInstall(args, cwd)` helper (analogous to `runGsdTools`) that calls `node INSTALL_PATH`:

```js
function runInstall(args, cwd) {
  try {
    const result = execSync('node "' + INSTALL_PATH + '" ' + args, {
      cwd, encoding: 'utf-8', stdio: ['pipe', 'pipe', 'pipe'],
    });
    return { success: true, output: result.trim() };
  } catch (err) {
    return { success: false, output: err.stdout?.toString().trim() || '', error: err.stderr?.toString().trim() || err.message };
  }
}
```

Tests:
```
TC-CB-1: --disable-breaker writes disabled:true to circuit-breaker-state.json
TC-CB-2: --enable-breaker after disable writes disabled:false
TC-CB-3: --enable-breaker when no state file exits 0 without error
```

For TC-CB-1: run disable-breaker with `cwd: tmpDir`, read `path.join(tmpDir, '.claude', 'circuit-breaker-state.json')`, assert `state.disabled === true`.
For TC-CB-2: chain disable then enable, assert `state.disabled === false`.
For TC-CB-3: assert command exits 0 (no throw), check output contains 'enabled'.

Test isolation: each test uses fresh tmpDir via `beforeEach`/`afterEach`.
