# Plan v0.14-02-03 Summary: Gap Closure — Guard Drift Enforcement + Fixture-Based Drift Tests

## What Was Built

Closed three verifier-identified gaps from phase v0.14-02:

### TASK-GC-01: Check 5 — Guard Drift Enforcement

Added a new Check 5 to `bin/check-spec-sync.cjs` that enforces bidirectional consistency between XState guard names and `formal/tla/guards/qgsd-workflow.json`:

- **Forward check (drift detection):** Every guard in the XState machine (`minQuorumMet`, `noInfiniteDeliberation`, `phaseMonotonicallyAdvances`) must appear as a key in `guards/qgsd-workflow.json`. If a guard is renamed in the XState machine and the JSON is not updated, `npm test` now fails.
- **Reverse check (orphan detection):** Every key in `guards/qgsd-workflow.json` must correspond to an active guard in the XState machine. Removes orphaned mapping entries from causing silent drift.
- **Corroboration:** Check 5 also scans `QGSDQuorum.tla` source text for camelCase guard name occurrences in comment documentation.
- **`--guards-path` CLI flag:** Added to `check-spec-sync.cjs` for fixture-based testing; allows tests to pass a temp guards JSON without modifying the repo.

Also added `--tla-path` CLI flag for fixture-based TLA+ substitution (needed for TASK-GC-03 test).

### TASK-GC-02: Extended Check 4 — Alloy Guard Name Scan

Extended the existing Check 4 (Alloy orphan scan) to also scan `.als` files for verbatim XState camelCase guard name occurrences. Reports which XState guard names appear in Alloy source (informational, `ok()` level). Design decision: Alloy guard orphan detection remains advisory (`warn()`/`ok()`) not `fail()` because Alloy models may abstract guard logic under different predicate names — the authoritative check is Check 5 against `guards/qgsd-workflow.json`.

### TASK-GC-03: Fixture-Based Drift Injection Tests

Replaced the empty no-op test 2 (52 lines, 0ms execution, 0 assertions) with two real fixture-based tests:

1. **State drift test:** Reads `formal/tla/QGSDQuorum.tla`, injects `"PHANTOM_STATE"` into the TypeOK `phase \in {...}` block via regex substitution, writes to a temp file, invokes `check-spec-sync.cjs --tla-path=<tmpfile>`, asserts exit code 1 and that the output mentions PHANTOM_STATE. Executes in ~84ms.

2. **Guard drift test (Copilot improvement):** Reads `formal/tla/guards/qgsd-workflow.json`, injects `"phantomGuardXYZ": "FALSE"` as an orphaned mapping, writes to a temp file, invokes `check-spec-sync.cjs --guards-path=<tmpfile>`, asserts exit code 1 and that the output mentions phantomGuardXYZ. Executes in ~81ms.

Both tests use `fs.mkdtempSync` + `fs.rmSync` cleanup via try/finally.

## Decisions

- `--tla-path` and `--guards-path` CLI overrides added to `check-spec-sync.cjs` — cleanest approach for fixture isolation; no temp-file swap risk, no env var needed
- Alloy guard orphan detection remains `ok()` (informational) not `fail()` — Alloy models are intentional abstractions that may not use XState camelCase guard names verbatim

- Guard drift test added at Copilot's suggestion (quorum improvement incorporated into plan)
