Create bin/parse-tlc-counterexample.cjs with these exports:

1. `parseITFTrace(traceJsonPath)` — Reads an ITF JSON file (as output by TLC `-dumpTrace json`). Returns:
   ```
   {
     states: Array<Object>,     // Each state is a plain object of variable->value
     loopPoint: number | null,  // Lasso cycle index (from ITF "loop" field), null if none
     violated_invariant: string | null,  // From ITF "violated_invariant" or "violated"
     trace_length: number       // states.length
   }
   ```
   Handles edge cases: empty file (throw with clear message), malformed JSON (throw), missing states array (return empty states).

2. `normalizeITFValue(value)` — Handles ITF special value encoding:
   - `{ "#bigint": "123" }` -> `BigInt(123)` or string "123" (use string to keep JSON-serializable)
   - `{ "#set": [1,2,3] }` -> `[1,2,3]` (sorted array)
   - `{ "#map": [["k1","v1"],["k2","v2"]] }` -> `{ k1: "v1", k2: "v2" }` (plain object)
   - Primitive values pass through unchanged
   - Recursively normalize nested objects/arrays

3. `extractStateFields(states, fieldNames?)` — Given parsed states array and optional field name filter, returns filtered states containing only the named fields. If fieldNames is null/undefined, returns all fields. Useful for focusing diffs on relevant variables.

- parseITFTrace extracts violated_invariant

- normalizeITFValue converts bigint encoding to string
- normalizeITFValue converts set encoding to sorted array
- normalizeITFValue converts map encoding to plain object
- normalizeITFValue passes primitives through
- extractStateFields filters to named fields only
- extractStateFields returns all fields when no filter

Create bin/diagnostic-diff-generator.cjs with these exports:

1. `generateStateDiff(statesA, statesB, options?)` — Compares two state sequence arrays (as returned by parseITFTrace). Uses json-diff-ts `diff()` function to compare corresponding states index by index. Returns:
   ```
   {
     aligned_length: number,       // min(statesA.length, statesB.length)
     length_mismatch: boolean,     // true if different lengths
     per_state_diffs: Array<{
       index: number,
       changes: Array<{ key, oldValue, newValue }>
     }>,
     summary: {
       total_changes: number,
       changed_fields: string[],   // unique field names that differ
       first_divergence_index: number | null  // first state index with diffs
     }
   }
   ```
   Options: `{ fieldFilter?: string[] }` — if provided, only compare these fields (calls extractStateFields internally by importing from parse-tlc-counterexample.cjs).

2. `formatDiffAsMarkdown(diffResult)` — Takes generateStateDiff output and formats as human-readable markdown:
   ```
   ## State Divergence Report

   **Traces diverge at state {N}** ({M} total changes across {K} fields)

   ### State {index}
   | Field | Trace A | Trace B |
   |-------|---------|---------|
   | timeout | 5000 | 10000 |
   ```
   Empty diff = "Traces are identical" message.

Import `parseITFTrace` and `extractStateFields` from `./parse-tlc-counterexample.cjs` — do NOT re-implement parsing logic. Import `diff` from `json-diff-ts`.
