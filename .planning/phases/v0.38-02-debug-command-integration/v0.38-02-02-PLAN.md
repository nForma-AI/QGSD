**Step E modification (Render NEXT STEP table):**

Update the result table template to include a FORMAL row between the worker rows and the CONSENSUS row. The table in Step E currently shows:

```
| Model        | Confidence   | Next Step                                   |
|--------------|--------------|---------------------------------------------|
| Gemini       | [confidence] | [next_step]                                 |
| ...          |              |                                             |
| CONSENSUS    | [HIGH/MED/-] | [consensus step or "No consensus"]          |
```

Insert a FORMAL row before CONSENSUS:

```
| FORMAL       | [verdict-confidence] | [verdict-description]                |
```

Where:
- If $FORMAL_VERDICT is "reproduced": confidence = "HIGH (model)", description = "Model {name} reproduced bug. Constraint: {top constraint text}" (use first constraint from $FORMAL_CONTEXT.constraints, or "see model trace" if no constraints)
- If $FORMAL_VERDICT is "not-reproduced": confidence = "LOW (model)", description = "Models exist but none reproduced. Bug may be environmental or outside model scope."
- If $FORMAL_VERDICT is "no-model": confidence = "N/A", description = "No formal model covers this failure. Gap tracked for future model creation."

Also add a note after the table when FORMAL and CONSENSUS diverge:
```
If FORMAL verdict is "reproduced" but CONSENSUS root cause differs from model trace:
  **Note:** Formal model reproduction suggests state invariant violation, but workers attribute to a different cause. Consider: is the invariant too strict, or is the worker diagnosis incomplete?
```

**Step G modification (Execute or escalate):**

After the existing consensus-handling logic in Step G, add a conditional for gap persistence:

```
### Gap Persistence (formal model coverage tracking)

If $FORMAL_VERDICT is "no-model":
  Read .planning/formal/bug-model-gaps.json (create with { "gaps": [] } if missing).
  Generate bug_id: first 8 chars of SHA-256 hash of $ARGUMENTS.
  Check if an entry with the same bug_id already exists — if so, skip (no duplicate).
  Append entry:
    {
      "bug_id": "<computed hash>",
      "description": "$ARGUMENTS" (truncated to 500 chars),
      "failure_context": "$BUNDLE" (truncated to 500 chars),
      "timestamp": "<ISO 8601>",
      "status": "no_coverage",
      "worker_consensus_root_cause": "<consensus root cause or 'no consensus'>",
      "worker_consensus_next_step": "<consensus next step or 'no consensus'>"
    }
  Write updated JSON back to .planning/formal/bug-model-gaps.json.
  Log: "Gap tracked: bug_id={bug_id} — no formal model coverage for this failure."
```

Use the existing `hashBugId` pattern from `bin/formal-scope-scan.cjs` (SHA-256, 8-char hex) for consistency with Phase v0.38-01.

**Important:**
- The FORMAL row MUST always appear in the table (even when verdict is "no-model") — per research recommendation, it signals "we checked" vs "we didn't check"
- Gap persistence MUST happen in Step G (not Step A.5) because it needs the worker consensus results
- Use fail-open: if bug-model-gaps.json read/write fails, log to stderr and continue
