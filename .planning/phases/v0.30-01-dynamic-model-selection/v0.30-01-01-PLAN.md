Create `bin/task-classifier.cjs` as a CommonJS module with `'use strict'` at top. Zero new dependencies.

**COMPLEXITY_MAP constant:**
```
trivial:  { tier: 'haiku',  thinking_budget: 0,     description: 'formatting, typos, simple renames' }
simple:   { tier: 'sonnet', thinking_budget: 8000,   description: 'single-file edits, config changes, test fixes' }
moderate: { tier: 'sonnet', thinking_budget: 16000,  description: 'multi-file changes, refactoring, feature additions' }
complex:  { tier: 'opus',   thinking_budget: 31999,  description: 'architecture decisions, new subsystems, formal verification' }
```

**classifyTask(taskEnvelope) function:**
Takes a task envelope object (or null). Returns one of: 'trivial', 'simple', 'moderate', 'complex'.
Heuristic signals:
- `fileCount`: from `(taskEnvelope.files || []).length`
- `hasArchitecture`: regex `/architect|design|system|interface|api/i` on objective
- `isExploration`: regex `/explore|research|investigate|discover/i` on objective
- `isReview`: regex `/review|check|audit|inspect/i` on objective
- `riskLevel`: from `taskEnvelope.risk_level` (default 'medium')
- `planTaskCount`: from `taskEnvelope.task_count` (default 1)

Classification rules (ordered by specificity):
1. null/undefined envelope -> 'moderate' (fail-open default)
2. isExploration -> 'trivial'
3. isReview AND fileCount <= 3 -> 'simple'
4. riskLevel === 'high' OR hasArchitecture -> 'complex'
5. fileCount > 5 OR planTaskCount > 3 -> 'moderate'
6. riskLevel in ['low', 'routine'] -> 'simple'
7. default -> 'moderate'

**getModelRecommendation(complexity, config) function:**
Returns `{ complexity, tier, thinking_budget, description }`.
Respects `config.model_routing` overrides for tier (e.g., `config.model_routing.simple = 'haiku'`).
Respects `config.thinking_budget_exploration`, `config.thinking_budget_review`, `config.thinking_budget_architecture` overrides for budget.
Falls back to COMPLEXITY_MAP defaults.

**readTaskEnvelope(cwd) function:**
Reads `.planning/task-envelope.json` if it exists, returns parsed JSON or null. Wrapped in try/catch (fail-open).

Export: `{ classifyTask, getModelRecommendation, readTaskEnvelope, COMPLEXITY_MAP }`

**Config keys** -- add to DEFAULT_CONFIG in `hooks/config-loader.js` (FLAT keys, not nested -- shallow merge would clobber nested objects):
```javascript

thinking_budget_review: 8000,
thinking_budget_architecture: 31999,
model_routing_enabled: true,
model_routing_cooldown_rounds: 3,
model_routing: {},  // per-complexity tier overrides, e.g. { simple: 'haiku' }
```

Modify `hooks/nf-prompt.js` to inject thinking budget directive and model tier recommendation into quorum dispatch.

**Integration point:** Inside the quorum injection path (after quorum instructions are built, before they are written to additionalContext). This is in the main stdin handler where `isQuorumCommand` is true.

**Steps:**
1. At top of file, add: `const taskClassifier = (() => { try { return require(path.join(__dirname, '..', 'bin', 'task-classifier.cjs')); } catch { return null; } })();`
   Use lazy require wrapped in try/catch -- fail-open if classifier not found.

2. In the quorum injection path, after building `quorumInstructions` string but before writing output:
   ```javascript
   // Task complexity classification + thinking budget directive
   if (taskClassifier && config.model_routing_enabled !== false) {
     const envelope = taskClassifier.readTaskEnvelope(cwd);
     const complexity = taskClassifier.classifyTask(envelope);
     const recommendation = taskClassifier.getModelRecommendation(complexity, config);

     // Inject thinking budget directive
     const thinkingDirective = `\n\nTHINKING BUDGET: Task classified as "${recommendation.complexity}" (${recommendation.description}). ` +
       `Use maximum ${recommendation.thinking_budget} thinking tokens. ` +
       `Model tier recommendation: ${recommendation.tier}.`;
     quorumInstructions += thinkingDirective;

     // Write classification to task-envelope for downstream consumers
     try {
       const envPath = path.join(cwd, '.planning', 'task-classification.json');
       fs.writeFileSync(envPath, JSON.stringify({
         ts: new Date().toISOString(),
         complexity: recommendation.complexity,
         tier: recommendation.tier,
         thinking_budget: recommendation.thinking_budget,
       }, null, 2) + '\n', 'utf8');
     } catch { /* fail-open */ }
   }
   ```

3. After editing `hooks/nf-prompt.js`, copy to `hooks/dist/nf-prompt.js`.

4. Run `node bin/install.js --claude --global` to sync installed hooks.

**Critical:** Do NOT change the existing quorum dispatch logic (slot selection, fan-out, scoring). Only APPEND the thinking budget directive to `quorumInstructions` and write the classification side-file. The model tier recommendation is advisory -- it does not programmatically switch the model for slots (that would require changes to the MCP dispatch which is out of scope for this advisory approach).

**Anti-pattern to avoid:** Do NOT add a new hook or modify the hook event type. All changes go into the existing `nf-prompt.js` UserPromptSubmit handler.
