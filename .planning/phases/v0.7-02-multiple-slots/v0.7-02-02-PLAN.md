<objective>
Extend the "Add new agent" flow in `commands/qgsd/mcp-setup.md` to support native CLI slots beyond the first instance (`copilot-2`, `opencode-2`, `codex-cli-2`, `gemini-cli-2`). After a new native CLI slot is written to `~/.claude.json`, immediately append it to `quorum_active` in `qgsd.json`. This closes MULTI-02 (wizard support for N-instance native CLI families) and MULTI-03 (wizard path for adding new slots).
</objective>

<context>
The existing "Add new agent" template list in `mcp-setup.md` only shows `claude-1..5`. These are all claude-mcp-server instances with ANTHROPIC credentials.

Native CLI agents (copilot, opencode, codex, gemini) use their own binary and auth mechanism:
- `copilot-mcp-server` — GitHub Copilot auth (no ANTHROPIC_API_KEY)
- `opencode-mcp-server` — OpenCode config (no ANTHROPIC_API_KEY)
- `codex-mcp-server` — Codex/OpenAI auth (different env var pattern)
- `gemini-mcp-server` — Gemini auth (no ANTHROPIC_API_KEY)

The mcpServers entry for a second native CLI slot copies the binary path from the first existing slot of that family. No ANTHROPIC credentials are written. The `command` and `args[0]` are identical to the first slot; only the key name differs (`codex-cli-2` vs `codex-cli-1`).

After creating the entry, the apply step must also call `addSlotToQuorumActive(newSlotName, qgsdJsonPath)` (built in Plan 01) to append the new slot to `quorum_active`.

The template list must show only non-configured native CLI slots (e.g., if `codex-cli-1` exists but `codex-cli-2` doesn't, offer `codex-cli-2`). If both already exist, neither is shown (or shown as "already configured").
</context>

If "Add and start":

1. Backup `~/.claude.json`:
```bash
cp ~/.claude.json ~/.claude.json.backup-$(date +%Y-%m-%d-%H%M%S) 2>/dev/null || true
```

2. Resolve binary path from existing first slot:
```bash
NATIVE_MCP_PATH=$(node -e "
const fs = require('fs'), os = require('os');
try {
  const cj = JSON.parse(fs.readFileSync(os.homedir() + '/.claude.json', 'utf8'));
  const src = cj.mcpServers[process.env.SOURCE_SLOT];
  if (src && src.args && src.args[0]) { process.stdout.write(src.args[0]); }
} catch(e) {}
" SOURCE_SLOT="{source-slot-name}")
```

3. Copy mcpServers entry (command + args + env) from source slot to new slot:
```bash
node -e "
const fs   = require('fs');
const os   = require('os');
const claudeJsonPath = os.homedir() + '/.claude.json';
let cj = {};
try { cj = JSON.parse(fs.readFileSync(claudeJsonPath, 'utf8')); } catch(e) {}
const srcSlot = process.env.SOURCE_SLOT;
const newSlot = process.env.NEW_SLOT;
const src = (cj.mcpServers || {})[srcSlot];
if (!src) { process.stdout.write(JSON.stringify({ written: false, error: 'source slot not found' }) + '\n'); process.exit(1); }
cj.mcpServers = cj.mcpServers || {};
cj.mcpServers[newSlot] = JSON.parse(JSON.stringify(src)); // deep copy
fs.writeFileSync(claudeJsonPath, JSON.stringify(cj, null, 2));
process.stdout.write(JSON.stringify({ written: true, newSlot, sourceSlot: srcSlot }) + '\n');
" SOURCE_SLOT="{source-slot-name}" NEW_SLOT="{new-slot-name}"
```

4. Append new slot to `quorum_active` in qgsd.json:
```bash
node -e "
const fs = require('fs'), os = require('os');
const qgsdPath = os.homedir() + '/.claude/qgsd.json';
let cfg = {};
try { cfg = JSON.parse(fs.readFileSync(qgsdPath, 'utf8')); } catch(e) {}
const active = Array.isArray(cfg.quorum_active) ? cfg.quorum_active : [];
const newSlot = process.env.NEW_SLOT;
if (!active.includes(newSlot)) {
  cfg.quorum_active = [...active, newSlot];
  fs.writeFileSync(qgsdPath, JSON.stringify(cfg, null, 2) + '\n');
  process.stdout.write(JSON.stringify({ added: true, slot: newSlot }) + '\n');
} else {
  process.stdout.write(JSON.stringify({ added: false, slot: newSlot, reason: 'already present' }) + '\n');
}
" NEW_SLOT="{new-slot-name}"
```

5. Invoke `/qgsd:mcp-restart {new-slot-name}` to start the new agent process.
