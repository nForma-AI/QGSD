<feature>
  <name>task-envelope.cjs — init, update, read, validate</name>
  <files>bin/task-envelope.cjs, bin/task-envelope.test.cjs</files>
  <behavior>
    Schema: { schema_version: "1", phase: "v0.18-03", created_at: ISO8601, risk_level: "low"|"medium"|"high", research: { objective, constraints, target_files, confidence, summary }, plan: { plan_path, key_decisions, wave_count } }

    init command:
    - Required: --phase <phase-id>
    - Optional: --objective, --constraints, --risk-level (default "medium"), --target-files (comma-separated), --confidence
    - Validates risk_level against ["low","medium","high"]; invalid → stderr warning, use "medium"
    - Writes new envelope with research section; exits 1 if phase dir not found
    - Atomic write: tmpPath = `${targetPath}.${Date.now()}.tmp`; writeFileSync(tmpPath); renameSync(tmpPath, targetPath)

    update --section plan command:
    - Required: --phase, --plan-path
    - Optional: --key-decisions (comma-separated), --wave-count
    - Reads existing envelope; merges plan section; preserves ALL other sections unchanged
    - If envelope doesn't exist: creates minimal envelope then adds plan section
    - Atomic write same pattern

    read command:
    - Required: --phase
    - Prints JSON to stdout; exits 1 if envelope not found

    validate command:
    - Required: --phase
    - Returns JSON { valid: bool, errors: string[] } to stdout
    - Errors include: invalid schema_version, invalid risk_level, missing required fields

    Exported functions (for unit tests):
    - validateEnvelope(obj) → { valid: bool, errors: string[] }
    - ENVELOPE_SCHEMA (const object describing required fields)

    Cases (input → output):
    - validateEnvelope({ schema_version: "1", phase: "v0.18-03", risk_level: "medium", research: { objective: "test" } }) → { valid: true, errors: [] }
    - validateEnvelope({ schema_version: "2", phase: "v0.18-03", risk_level: "medium" }) → { valid: false, errors: ["schema_version must be '1'"] }
    - validateEnvelope({ schema_version: "1", phase: "invalid!", risk_level: "high" }) → { valid: false, errors: ["phase must match v\\d+\\.\\d+-\\d{2}"] }
    - validateEnvelope({ schema_version: "1", phase: "v0.18-03", risk_level: "CRITICAL" }) → { valid: false, errors: ["risk_level must be low|medium|high"] }
    - init with risk-level "CRITICAL" → stderr warning, file written with risk_level "medium"
    - update --section plan on existing envelope → research section preserved, plan section added
    - update --section plan on missing envelope → creates envelope, writes plan section
  </behavior>
