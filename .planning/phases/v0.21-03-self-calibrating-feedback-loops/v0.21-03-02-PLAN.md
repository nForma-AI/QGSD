```javascript
// ── LOOP-01: export-prism-constants pre-step ─────────────────────────────────
// Ensures rates.const is always current from the scoreboard before PRISM runs.
// Fail-open: if export fails, run-prism continues with whatever rates.const exists.
{
  const exportConstantsPath = path.join(__dirname, 'export-prism-constants.cjs');
  const exportResult = spawnSync(process.execPath, [exportConstantsPath], {
    encoding: 'utf8',
    cwd: process.cwd(),
    timeout: 10000,
  });
  if (exportResult.status !== 0 || exportResult.error) {
    process.stderr.write(
      '[run-prism] Warning: export-prism-constants pre-step failed — rates.const may be stale.\n' +
      (exportResult.stderr || '') + '\n'
    );
  } else {
    process.stdout.write('[run-prism] Pre-step: rates.const updated from scoreboard.\n');
  }
}
// ─────────────────────────────────────────────────────────────────────────────
```

**Key constraints:**
- Use `process.execPath` (not `'node'`) as the executable — this is the pattern used throughout QGSD's spawnSync calls (e.g., in run-prism.test.cjs itself)
- Use `path.join(__dirname, 'export-prism-constants.cjs')` to locate the script — `__dirname` is already defined at the top of run-prism.cjs as the `bin/` directory
- Set `timeout: 10000` (10 seconds) on the spawnSync call — the research anti-patterns explicitly state to cap the pre-step at 10s
- Wrap in a block `{...}` to avoid polluting the outer scope with `exportResult`
- Do NOT move or change any existing logic — only insert the new block
- Do NOT make the pre-step abort PRISM on failure — the `if` branch warns and continues (fail-open)
