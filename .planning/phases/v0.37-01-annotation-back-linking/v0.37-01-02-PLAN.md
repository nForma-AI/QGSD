Current pattern (line 2067-2068):
```javascript
hasReqAnnotation = /@req\s+[A-Z]+-\d+/i.test(content) ||
                   /\/\/\s*req:\s*[A-Z]+-\d+/i.test(content);
```

Replace with expanded pattern that also matches `@requirement`:
```javascript
hasReqAnnotation = /@req(?:uirement)?\s+[A-Z]+-\d+/i.test(content) ||
                   /\/\/\s*req(?:uirement)?:\s*[A-Z]+-\d+/i.test(content) ||
                   /\/\/\s*Requirements:\s*[A-Z]+-\d+/i.test(content);
```

This ensures:
- `// @requirement TLINK-01` matches (the new pattern from this milestone)
- `// @req TLINK-01` still matches (backward compat)
- `// req: TLINK-01` still matches (backward compat)
- `// Requirements: TLINK-01, TLINK-02` matches (header comment style)

Modify the return value of sweepTtoR() in bin/nf-solve.cjs to include an `annotation_coverage_percent` field in the detail object.

Implementation:
1. Before the return statement (around line 2114), calculate:
   - `annotatedCount`: Number of test files that have a `@requirement` or `@req` annotation (regardless of whether they were also mapped by other means like code-trace-index or formal-test-sync)
   - To get this, add a counter during the main loop that increments when `hasReqAnnotation` is true (move the annotation check to run for ALL files, not just those not already mapped by code-trace-index)
2. Calculate: `annotation_coverage_percent = Math.round((annotatedCount / testFiles.length) * 100)`
3. Add to the detail return object:
```javascript
return {
  residual: orphanItems.length,
  detail: {
    orphan_tests: orphanItems,
    total_tests: testFiles.length,
    mapped: mapped,
    annotation_coverage_percent: annotation_coverage_percent,
    annotated_count: annotatedCount,
    scoped: focusSet ? false : undefined,
  },
};
```

Important: The annotation check must run for ALL test files to get accurate coverage, not just files that aren't already mapped by other mechanisms. This means moving the annotation regex check to happen unconditionally (before the code-trace-index early continue), storing the result, then using it both for mapping decisions and for the final coverage count.
