  <action>
Create a new CLI tool `bin/annotate-tests.cjs` that scans test files, queries the proximity graph, and outputs @requirement annotation suggestions.

CLI interface:
```
node bin/annotate-tests.cjs                    # Scan default test dirs (bin/, test/)
node bin/annotate-tests.cjs --dir test/        # Scan specific directory
node bin/annotate-tests.cjs --write            # Auto-inject annotations into files
node bin/annotate-tests.cjs --json             # JSON output (default is human-readable)
```

Implementation steps:

1. **Discover test files**: Scan `bin/*.test.cjs` and `test/*.test.cjs` (or --dir if provided). Collect array of `{ file, absPath }`.

2. **Filter already-annotated**: For each test file, read content and check for existing `@requirement` or `@req` annotations. Skip files that already have annotations (per pitfall 2 from research: do NOT overwrite existing annotations).

3. **Load proximity graph**: Read `.planning/formal/proximity-index.json`. If missing, exit with message "Run `node bin/formal-proximity.cjs` first to build proximity index."

4. **Query proximity for suggestions**: For each unannotated test file:
   - Build node key: `code_file::` + relative path (e.g., `code_file::test/foo.test.cjs`)
   - Use `require('./formal-query.cjs').reach(pi, nodeKey, 3, ['requirement'])` to find reachable requirement nodes
   - Score candidates by proximity (closer hops = higher confidence)
   - Select best match (highest score) with minimum threshold 0.4
   - If no node key exists in pi, try filename-based fuzzy matching (look for nodes containing the test filename)

5. **Output suggestions**: For each file with a suggestion:
   ```
   test/foo.test.cjs:
     Suggested: // @requirement REQ-01 (confidence: 0.7)
   ```

6. **--write mode**: When --write is passed, inject the annotation as a comment on line 1 of the file (after shebang if present). Format: `// @requirement REQ-ID` on its own line. Only write if confidence >= 0.5.

7. **--json mode**: Output JSON array:
   ```json
   [{ "file": "test/foo.test.cjs", "suggested_requirement": "REQ-01", "confidence": 0.7, "recommendation": "// @requirement REQ-01" }]
   ```

8. **Summary line**: At end, print: `N suggestions for M orphan test files (K already annotated)`

Error handling: fail-open on file read errors, warn on missing proximity nodes, exit 0 even if no suggestions found.
