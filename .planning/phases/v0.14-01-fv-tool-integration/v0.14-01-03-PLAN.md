# Plan v0.14-01-03 — Wire INTG-04: run-formal-verify calls xstate-to-tla as first step

## Goal

Satisfy INTG-04: `run-formal-verify.cjs` must call `xstate-to-tla.cjs` as its spec generation step (pipeline is end-to-end wired). Currently STEPS[0] invokes `generate-formal-specs.cjs`. This plan resolves the wiring gap, runs `npm test`, and writes SUMMARY.md + updates STATE.md.

## Context

Current STEPS[0] in `run-formal-verify.cjs`:
```js
{
  tool: 'generate', id: 'generate:quorum-specs',
  label: 'Generate quorum specs from XState machine (TLA+, cfg, Alloy, PRISM)',
  type: 'node', script: 'generate-formal-specs.cjs', args: [],
},
```

`generate-formal-specs.cjs` does its own regex-based XState parsing — it does NOT call `xstate-to-tla.cjs` internally. The two scripts are parallel implementations:
- `generate-formal-specs.cjs`: regex-based, generates TLA+/Alloy/PRISM from XState via text pattern matching
- `xstate-to-tla.cjs`: esbuild-based, compiles the TypeScript machine file and walks machine.config — generates TLA+ + cfg only

INTG-04 requires the pipeline to call `xstate-to-tla.cjs` as the spec generation step. The correct resolution is:

**Decision**: Update STEPS[0] to call `xstate-to-tla.cjs` with the canonical machine file arg (`src/machines/qgsd-workflow.machine.ts`). Keep `generate-formal-specs.cjs` as a separate step labeled `generate:alloy-prism-specs` (since xstate-to-tla.cjs only generates TLA+/cfg, not Alloy/PRISM). This makes the pipeline fully correct:
- Step 0 (generate:tla-from-xstate): `xstate-to-tla.cjs src/machines/qgsd-workflow.machine.ts --module=QGSDQuorum --config=.formal/tla/guards/qgsd-workflow.json`
- Step 1 (generate:alloy-prism-specs): `generate-formal-specs.cjs` (generates Alloy + PRISM models)
