# Phase v0.25-01 Plan 03: Requirement Traceability Wiring Summary

**One-liner:** Centralized requirement map with 28 check_ids wired to all 18 formal verification runners via `getRequirementIds()` so every NDJSON check result now includes `requirement_ids`.

## What Was Built

### Task 1: bin/requirement-map.cjs (committed 53ba42d6)

Created the centralized requirement map as a CommonJS module exporting:
- `CHECK_ID_TO_REQUIREMENTS` — the 28-entry object literal mapping each check_id to its requirement ID array
- `getRequirementIds(checkId)` — fail-open lookup that returns a copy of the array (or `[]` for unknown IDs)

The 28 check_ids cover all surfaces: TLA+ (14), Alloy (7), PRISM (3), UPPAAL (1), CI (3).

### Task 2: Wire all 18 runners (committed 598bae18)

Added `require('./requirement-map.cjs')` and `requirement_ids: getRequirementIds(check_id)` to every `writeCheckResult()` call across 18 runner files:

| Surface | Runners |
|---------|---------|
| TLA+ | run-tlc, run-protocol-tlc, run-breaker-tlc, run-oscillation-tlc, run-account-manager-tlc, run-stop-hook-tlc |
| Alloy | run-alloy, run-audit-alloy, run-transcript-alloy, run-installer-alloy, run-account-pool-alloy, run-quorum-composition-alloy |
| PRISM | run-prism, run-oauth-rotation-prism |
| UPPAAL | run-uppaal |
| CI | check-trace-redaction, check-trace-schema-drift, check-liveness-fairness |

All 18 files pass `node --check` syntax validation.

### Task 3: bin/requirement-map.test.cjs (committed a14fe73d)

27 test cases in 7 describe blocks:
- Module shape (exports, count, value types)
- Fail-open for unknown/empty/null/undefined inputs
- TLA+ runners spot-checks (tla:quorum-safety, tla:breaker, tla:stop-hook, etc.)
- Alloy runners spot-checks (alloy:scoreboard, alloy:transcript, alloy:account-pool, etc.)
- PRISM and UPPAAL runners spot-checks
- CI runners spot-checks
- Return type consistency (always array, mutation safety)

All 27 tests pass: `node --test bin/requirement-map.test.cjs`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] getRequirementIds returned shared array reference**
- **Found during:** Task 3 test run
- **Issue:** For known check_ids, `getRequirementIds` returned the array from the object literal directly. Callers could mutate it (e.g., `result.push(...)`) and corrupt the shared source-of-truth.
- **Fix:** Changed to `found.slice()` to return a copy. The `|| []` path already returned a new array.
- **Files modified:** bin/requirement-map.cjs
- **Commit:** a14fe73d (included in Task 3 commit)

### Discovery: check_id name mismatches

The plan proposed longer check_id names that did not match what runners actually emit:

| Plan proposed | Actual runner check_id |
|---|---|
| alloy:scoreboard-recompute | alloy:scoreboard |
| alloy:availability-parsing | alloy:availability |
| alloy:transcript-scan | alloy:transcript |
| alloy:account-pool-structure | alloy:account-pool |
| ci:liveness-fairness | ci:liveness-fairness-lint |

The requirement map was created with the actual runner check_ids. This is a documentation mismatch in the plan (not a code bug) — runners define the authoritative check_id strings.

### Scoping Decision: run-phase-tlc.cjs excluded

`bin/run-phase-tlc.cjs` does not call `writeCheckResult()` — it's a plan verification tool that runs TLA+ against planning truths, not a formal check result emitter. Correctly excluded from wiring.
