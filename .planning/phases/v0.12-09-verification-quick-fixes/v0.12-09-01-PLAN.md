# v0.12-09-01 — GAP-5: Fix MCconvergence workers bug

## Problem

`bin/run-oscillation-tlc.cjs` line 101:

```javascript
// MCoscillation has a PROPERTY (liveness) — use -workers 1 to avoid TLC multi-worker liveness bug.
// MCconvergence is safety-only (INVARIANT only) — 'auto' workers is safe.
const workers = configName === 'MCoscillation' ? '1' : 'auto';
```

`formal/tla/MCconvergence.cfg` line 11 declares:

```
PROPERTY ConvergenceEventuallyResolves
```

This is a **liveness PROPERTY**, not an INVARIANT-only spec. The comment is incorrect. TLC's
multi-worker liveness checking bug applies to MCconvergence equally. Running with `-workers auto`
on a multi-core machine risks a false-pass or silent skip of liveness verification.

**Source:** v0.12 milestone audit, GAP-5.

## Fix

Change `bin/run-oscillation-tlc.cjs` line 101:

```javascript
// Both MCoscillation and MCconvergence have PROPERTY (liveness) declarations.
// TLC has a known multi-worker liveness checking bug — always use -workers 1.
const workers = '1';
```

Remove the configName-dependent ternary. Both configs have liveness properties; the distinction
was based on an incorrect reading of MCconvergence.cfg.

Also update the comment block above (lines 99-101) to accurately describe the situation.

## Files Changed

- `bin/run-oscillation-tlc.cjs` — lines 99-101: replace ternary with unconditional `'1'`, update comment
