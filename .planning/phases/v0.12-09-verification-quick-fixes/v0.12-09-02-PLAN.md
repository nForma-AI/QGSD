# v0.12-09-02 — TLA-04: Add JAR conditional-skip guard to run-tlc.test.cjs

## Problem

`bin/run-tlc.test.cjs` test at line 24:

```
test('exits non-zero and prints download URL when tla2tools.jar is not found', ...)
```

This test assumes `.formal/tla/tla2tools.jar` is absent. However, `tla2tools.jar` is physically
present in `.formal/tla/` (gitignored, placed during development). When the JAR is present,
`run-tlc.cjs` passes the JAR check and either runs TLC or fails for a different reason —
it does **not** exit with the JAR-not-found error the test expects.

**Result:** Test fails with `AssertionError: 0 strictEqual 1` — exit code is 0, not 1.

**Root cause confirmed:** v0.12 audit shows 310 pass, 2 fail — this test is one of the 2.

The identical problem was correctly solved in phases 06-08 with a conditional-skip pattern:

```javascript
// from run-audit-alloy.test.cjs line 33-34
const jarPath = path.join(__dirname, '..', 'formal', 'alloy', 'org.alloytools.alloy.dist.jar');
if (fs.existsSync(jarPath)) { return; }  // skip — can't test absent-JAR path when JAR is present
```

## Fix

Add the same conditional-skip guard to `bin/run-tlc.test.cjs` in the JAR-not-found test.

**Step 1:** Add `const fs = require('fs');` import (not present in run-tlc.test.cjs).

**Step 2:** Inside the test body, after detecting `javaHome`, add:

```javascript
const jarPath = path.join(__dirname, '..', 'formal', 'tla', 'tla2tools.jar');
if (fs.existsSync(jarPath)) { return; }  // skip — can't test absent-JAR path when JAR is present
```

The `path` import is already present. The `javaHome` detection block already exists at lines 27-31.

## Files Changed

- `bin/run-tlc.test.cjs`:
  1. Add `const fs = require('fs');` after the existing imports (after line 10)
  2. Add JAR-existence skip guard inside the JAR-not-found test body (after javaHome detection)
