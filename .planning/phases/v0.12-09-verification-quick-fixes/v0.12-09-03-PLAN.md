# v0.12-09-03 ‚Äî ALY-02: Fix stale assertion name in run-alloy.cjs

## Problem

`bin/run-alloy.cjs` lines 113-116:

```javascript
process.stderr.write(
  '[run-alloy] WARNING: Counterexample found in NoSpuriousApproval assertion.\n' +
  '[run-alloy] This indicates a spec violation ‚Äî review quorum-votes.als.\n'
);
```

The assertion name `NoSpuriousApproval` is **stale**. It was the name from an earlier design.
The actual assertions in `formal/alloy/quorum-votes.als` are:

- `ThresholdPasses`
- `BelowThresholdFails`
- `ZeroApprovalsFail`

This means if a counterexample is ever found, the error message will direct the developer to
look for an assertion that does not exist.

Additionally, `VERIFICATION_TOOLS.md` line 62 references the stale name:

```
node bin/run-alloy.cjs  # checks NoSpuriousApproval assertion
```

**Also:** `run-alloy.test.cjs` may have the same issue. Check and fix if present.

## Fix

### 1. bin/run-alloy.cjs ‚Äî update error message

Change lines 113-116 from:

```javascript
process.stderr.write(
  '[run-alloy] WARNING: Counterexample found in NoSpuriousApproval assertion.\n' +
  '[run-alloy] This indicates a spec violation ‚Äî review quorum-votes.als.\n'
);
```

To:

```javascript
process.stderr.write(
  '[run-alloy] WARNING: Counterexample found in quorum-votes.als assertions\n' +
  '[run-alloy] (ThresholdPasses / BelowThresholdFails / ZeroApprovalsFail).\n' +
  '[run-alloy] This indicates a spec violation ‚Äî review quorum-votes.als.\n'
);
```
