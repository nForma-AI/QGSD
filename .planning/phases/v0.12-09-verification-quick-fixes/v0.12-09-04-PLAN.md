# v0.12-09-04 — PRM-02: Document manual aggregation step

## Problem

`bin/export-prism-constants.cjs` generates per-slot PRISM constant declarations:

```
const double tp_gemini = 0.85;
const double unavail_gemini = 0.15;
const double tp_opencode = 0.85;
const double unavail_opencode = 0.15;
const double tp_copilot = 0.85;
const double unavail_copilot = 0.15;
const double tp_codex = 0.85;
const double unavail_codex = 0.15;
```

But `.formal/prism/quorum.pm` uses **aggregate** constants:

```
const double tp_rate = 0.85;
const double unavail = 0.15;
```

These variable names are **incompatible**. You cannot paste `rates.const` directly into
`quorum.pm` or use the per-slot names as `-const` CLI flags — the model references
`tp_rate` and `unavail`, not `tp_gemini` etc.

The manual aggregation step (compute mean across slots) is undocumented anywhere.

**Decision:** Documentation approach preferred over renaming variables in
export-prism-constants.cjs (which would require updating tests and existing documented usage).

## Fix

Add a "Manual aggregation" subsection to the PRISM section of `VERIFICATION_TOOLS.md`,
immediately after the existing "Export empirical rates from scoreboard" block.

### Content to add

Insert after the existing PRISM run block (after the `-const tp_rate=... -const unavail=...`
example):

```markdown
**Note on variable names:** `rates.const` contains per-slot variables (`tp_gemini`,
`tp_opencode`, `tp_copilot`, `tp_codex`, and their `unavail_*` counterparts). `quorum.pm`
uses aggregate constants (`tp_rate`, `unavail`). A manual aggregation step is required:

```bash
# Example: compute aggregate tp_rate as mean of per-slot rates
# (adjust values from rates.const output)
tp_rate=$(node -e "console.log(((0.9 + 0.85 + 0.88 + 0.92) / 4).toFixed(4))")
unavail=$(node -e "console.log(((0.05 + 0.10 + 0.08 + 0.20) / 4).toFixed(4))")

$PRISM_BIN .formal/prism/quorum.pm -pf "P=? [ F s=1 ]" \
    -const tp_rate=$tp_rate -const unavail=$unavail
```

Replace the example values with the actual values from `.formal/prism/rates.const`.
```
