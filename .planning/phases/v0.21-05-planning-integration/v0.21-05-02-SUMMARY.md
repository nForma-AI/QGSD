# v0.21-05-02 Summary: TLC Verification Runner

## Status: COMPLETE

## What was built
Created `bin/run-phase-tlc.cjs` -- runs TLC on proposed-changes.tla generated from PLAN.md frontmatter and returns structured results with pass/violations/feedback fields. Supports `iterativeVerify` for single-attempt orchestration, `formatTlcFeedback` for human-readable planner feedback with truth-index mapping, and `runPhaseTlc` for raw TLC invocation.

## Key decisions
- TLA+ filename changed from `proposed-changes.tla` to `ProposedChanges.tla` to match MODULE name (TLC requires filename=module name)
- `jarOverride` option added to `runPhaseTlc` for testability (avoids global jar dependency in tests)
- Fail-open design: missing Java or tla2tools.jar returns structured error, never crashes
- `formatTlcFeedback` uses `/Req(\d{2})/` regex to parse zero-padded property names back to 0-based truth indices
