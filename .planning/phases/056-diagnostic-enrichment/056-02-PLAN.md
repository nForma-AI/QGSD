<objective>
Enrich reverse discovery (C->R and T->R) candidates with getCallers call counts so modules with 0 callers are flagged as likely dead code in the solve report.

Purpose: CREM-04 — sweepCtoR() and sweepTtoR() currently emit untraced files/tests as candidates without call-graph evidence. Adding caller_count from getCallersSync gives quorum reviewers an objective signal: a source module that is never called by anything is likely dead code, not a missing requirement. This prevents spurious C->R residuals from dead utility files.

Output:
- bin/nf-solve.cjs: sweepCtoR() and sweepTtoR() enrich their untraced candidates with caller_count (from getCallers) and dead_code_flag (true when caller_count === 0); fail-open when coderlm unavailable.
- Solve report output: C->R and T->R candidate listings show "(0 callers — likely dead code)" annotation for dead-code candidates.
</objective>

}
