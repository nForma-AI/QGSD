  <action>
Create `bin/repowise/hotspot.cjs`:

1. `#!/usr/bin/env node` shebang and `'use strict'`
2. Module header: `// bin/repowise/hotspot.cjs — Git-log churn scoring + heuristic complexity for Repowise hotspot detection`
3. Imports: `fs`, `path`, `{ spawnSync }` from `child_process`, `{ escapeXml }` from `./escape-xml.cjs`

4. DEFAULT_EXCLUDE_PATTERNS: array of regex patterns for noise paths:
   ```javascript
   const DEFAULT_EXCLUDE_PATTERNS = [
     /\/node_modules\//,
     /\/vendor\//,
     /\/dist\//,
     /\.min\.js$/,
     /\.min\.css$/,
     /package-lock\.json$/,
     /\.generated\./,
     /\/\.planning\//,
     /\/__snapshots__\//,
   ];
   ```

5. DEFAULT_MASS_REFACTOR_THRESHOLD = 50

6. `isExcluded(filePath, excludePatterns)` — returns true if filePath matches any exclude pattern

7. `parseGitNumstat(projectRoot, options)` — runs `git log --all --no-merges --numstat --diff-filter=AMD` via spawnSync (maxBuffer 50MB). Parses output into array of `{ sha, files: [{ path, added, deleted }] }`. Skip entries where files list length > massRefactorThreshold (but still record with weight).

8. `computeChurnScores(commits, options)` — takes parsed commits, returns `Map<path, churnScore>`. For each commit:
   - If fileCount >= massRefactorThreshold: weight = 1 / max(1, fileCount / threshold)
   - Otherwise: weight = 1
   - For each non-excluded file: churnScore += weight
   - Skip files matching exclude patterns

9. `estimateComplexity(filePath, projectRoot)` — reads file, counts non-blank non-comment lines:
   - Blank: trimmed line is empty
   - Comment: starts with `//`, `#`, `/*`, `*`, `--` (language-agnostic heuristic)
   - Returns line count (integer)

10. `normalizeMap(scoreMap)` — min-max normalize values to [0,1]. Returns new Map. If all values are equal, returns all 0.5.

11. `computeHotspots(projectRoot, options)` — main API:
    - options: `{ excludePatterns?, massRefactorThreshold?, since? }`
    - Parse git numstat
    - Compute churn scores
    - For each file with churn > 0: estimate complexity
    - Normalize both churn and complexity to [0,1]
    - Compute hotspot_score = normalized_churn * normalized_complexity
    - Return: `{ files: [{ path, churn, complexity, hotspot_score }], summary: { total_files, high_risk_count, medium_risk_count } }`
    - high_risk = hotspot_score > 0.7, medium_risk = hotspot_score > 0.4

12. `formatHotspotXml(hotspots)` — formats hotspot data as XML section for context-packer:
    ```xml
    <files>
      <file path="src/foo.js" churn="12" complexity="45" hotspot_score="0.85" risk="high"/>
      <file path="src/bar.ts" churn="5" complexity="30" hotspot_score="0.35" risk="low"/>
    </files>
    ```

13. CLI main function with `--project-root`, `--json`, `--exclude=pattern1,pattern2`, `--mass-refactor-threshold=N`, `--since=date`, `--help` flags

14. Exports: `module.exports = { computeHotspots, computeChurnScores, estimateComplexity, normalizeMap, isExcluded, parseGitNumstat, formatHotspotXml, DEFAULT_EXCLUDE_PATTERNS, DEFAULT_MASS_REFACTOR_THRESHOLD };`

15. `require.main === module` guard for CLI
