## Summary

This phase creates a single CLI dashboard script (`bin/cross-layer-dashboard.cjs`) that aggregates data from four existing sources (instrumentation-map.json for L1 coverage, gate-a-grounding.json for Gate A, gate-b-abstraction.json for Gate B, gate-c-validation.json for Gate C) and renders them in a terminal-friendly table. The phase also addresses INTG-04 by ensuring the dashboard reflects real-time state on each invocation.

All data sources already exist and produce well-defined JSON with consistent schemas. The gate scripts all support `--json` for machine-readable output. The nf-solve.cjs report format (box-drawing characters, padded columns, health indicators) establishes the project's terminal display convention. The dashboard should follow this same plain-text pattern -- no external dependencies, no ANSI color libraries.

**Primary recommendation:** Build a single `bin/cross-layer-dashboard.cjs` that reads the four JSON artifacts (re-running gate scripts to ensure freshness), computes a composite health view, and prints a formatted terminal report following the nf-solve.cjs display conventions.

### Pattern 1: Gate Data Aggregation via Child Process
**What:** Spawn each gate script with `--json` flag, parse stdout as JSON, aggregate into dashboard view.
**When to use:** When freshness guarantee is required (success criterion #2: "real-time state").
**Example:**
```javascript
// Source: nf-solve.cjs spawnTool pattern (lines 70-113)
function spawnGate(script) {
  const result = child_process.spawnSync(
    process.execPath,
    [path.join(__dirname, script), '--json'],
    { encoding: 'utf8', timeout: 30000, env: { ...process.env, PROJECT_ROOT: ROOT } }
  );
  if (result.status !== 0 && result.status !== 1) {
    return { ok: false, error: result.stderr };
  }
  try { return { ok: true, data: JSON.parse(result.stdout) }; }
  catch { return { ok: false, error: 'JSON parse failure' }; }
}
```

### Pattern 2: Cached Read with Freshness Check
**What:** Read existing JSON files directly, with optional `--refresh` flag to re-run gates first.
**When to use:** When speed matters more than guaranteed freshness (e.g., repeated dashboard checks during a session).
**Example:**
```javascript
// Default: read JSON files (fast, ~10ms)
// --refresh flag: re-run gate scripts first (slower, ~5-15s)
const REFRESH = process.argv.includes('--refresh');
if (REFRESH) {
  spawnGate('gate-a-grounding.cjs');
  spawnGate('gate-b-abstraction.cjs');
  spawnGate('gate-c-validation.cjs');
}
// Then read JSON files
const gateA = JSON.parse(fs.readFileSync(GATE_A_PATH, 'utf8'));
```

### Pattern 3: Terminal Table Following nf-solve Convention
**What:** Plain-text table with box-drawing characters, padded columns, and health indicators.
**When to use:** All terminal output in this project.
**Example:**
```javascript
// Source: nf-solve.cjs formatReport pattern (lines 2270-2392)
function healthIndicator(score, target) {
  if (score >= target) return '[PASS]';
  if (score >= target * 0.8) return '[WARN]';
  return '[FAIL]';
}

function formatDashboard(l1, gateA, gateB, gateC, maturity) {
  const lines = [];
  lines.push('Cross-Layer Alignment Dashboard');
  lines.push('=============================================');
  lines.push('');
  lines.push('Layer Health:');
  lines.push('---------------------------------------------');
  lines.push('L1 Coverage %:           ' + l1.toFixed(1).padStart(6) + '    ' + healthIndicator(l1/100, 0.5));
  lines.push('Gate A (Grounding):      ' + (gateA*100).toFixed(1).padStart(6) + '%   ' + healthIndicator(gateA, 0.8));
  lines.push('Gate B (Abstraction):    ' + (gateB*100).toFixed(1).padStart(6) + '%   ' + healthIndicator(gateB, 1.0));
  lines.push('Gate C (Validation):     ' + (gateC*100).toFixed(1).padStart(6) + '%   ' + healthIndicator(gateC, 0.8));
  lines.push('---------------------------------------------');
  // ...maturity summary...
  return lines.join('\n') + '\n';
}
```

### Anti-Patterns to Avoid
- **Importing gate modules directly:** Gate scripts have heavy initialization (XState machine loading, trace parsing). Use child_process spawn with `--json` for isolation.
- **ANSI color codes:** The project uses plain text with box-drawing characters. No chalk/colors dependency.
- **Computing scores in dashboard:** Dashboard is a VIEW, not a computation engine. All score computation lives in gate scripts.

## Common Pitfalls

### Pitfall 1: Stale Gate JSON Files
**What goes wrong:** Dashboard reads old JSON files that don't reflect recent changes (new traces, updated models).
**Why it happens:** Gate scripts write JSON to disk but only when explicitly run.
**How to avoid:** Default behavior should re-run gate scripts (spawn with `--json`). Offer `--cached` flag for fast reads when freshness isn't critical.
**Warning signs:** Dashboard shows different scores than `nf-solve --report-only`.

### Pitfall 2: Gate Script Exit Codes
**What goes wrong:** Gate B and Gate C exit with code 1 when target is NOT met. Dashboard treats this as an error.
**Why it happens:** `process.exit(result.target_met ? 0 : 1)` in gate scripts.
**How to avoid:** Dashboard must handle exit code 1 as "valid output, target not met" rather than "script failed". Check that stdout contains valid JSON regardless of exit code.
**Warning signs:** Dashboard shows "ERROR" for gates that are functioning correctly but below target.

### Pitfall 3: Missing Prerequisite Artifacts
**What goes wrong:** Gate scripts fail because upstream artifacts (failure-mode-catalog.json, test-recipes.json) don't exist yet.
**Why it happens:** If v0.29-03 hasn't been fully executed, L3 artifacts may not exist.
**How to avoid:** Dashboard should gracefully degrade -- show "N/A" for gates whose prerequisites are missing, rather than crashing.
**Warning signs:** Unhandled exceptions from gate scripts when run in a partially-complete v0.29 state.

### Pitfall 4: Confusing gate_maturity with gate scores
**What goes wrong:** Dashboard conflates gate enforcement level (ADVISORY/SOFT_GATE/HARD_GATE) with gate alignment scores (0-100%).
**Why it happens:** Both use the word "gate" but are orthogonal concepts.
**How to avoid:** Display them in clearly separated sections of the dashboard. Maturity is about per-model enforcement policy. Scores are about cross-layer alignment quality.

### spawnTool Pattern (from nf-solve.cjs)
```javascript
// Source: nf-solve.cjs lines 70-113
function spawnTool(relScript, args) {
  const script = path.isAbsolute(relScript) ? relScript : path.join(ROOT, relScript);
  const result = child_process.spawnSync(process.execPath, [script, ...args], {
    encoding: 'utf8',
    timeout: 60_000,
    env: { ...process.env, PROJECT_ROOT: ROOT },
    stdio: ['pipe', 'pipe', 'pipe'],
  });
  return {
    ok: result.status === 0 || result.status === 1, // gate scripts exit 1 when target not met
    status: result.status,
    stdout: (result.stdout || '').trim(),
    stderr: (result.stderr || '').trim(),
  };
}
```
