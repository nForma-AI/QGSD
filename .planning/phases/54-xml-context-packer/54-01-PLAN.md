<objective>
Build the core XML packing primitives for Repowise: an escapeXml() helper that safely handles all five XML-special characters, and a packFile() function that wraps file contents in `<file path="..." lang="...">...</file>` XML tags. These are the building blocks that context-packer.cjs (Plan 54-02) and all subsequent Repowise modules will compose.

Purpose: Establish the zero-dependency delivery format that all Repowise intelligence modules feed into. Without safe XML escaping, no packed output can be trusted; without packFile, there is no standard tag format.
Output: bin/repowise/escape-xml.cjs, bin/repowise/pack-file.cjs, and their tests.
</objective>

  <files>bin/repowise/escape-xml.cjs</files>
  <action>
Create `bin/repowise/escape-xml.cjs` with the following structure:

1. `#!/usr/bin/env node` shebang and `'use strict'` directive
2. Module header comment: `// bin/repowise/escape-xml.cjs — XML character escaping for Repowise context packing`
3. `escapeXml(str)` function:
   - Accepts a string, returns a string with XML-special characters replaced
   - Replacement order MUST be `&` first to prevent double-encoding:
     ```
     function escapeXml(str) {
       if (typeof str !== 'string') return '';
       return str
         .replace(/&/g, '&amp;')
         .replace(/</g, '&lt;')
         .replace(/>/g, '&gt;')
         .replace(/"/g, '&quot;')
         .replace(/'/g, '&apos;');
     }
     ```
   - Returns empty string for non-string inputs (defensive, not throwing)
4. Export: `module.exports = { escapeXml };`
5. No dependencies — pure function, no I/O, no side effects

Follow the code style of `bin/design-impact.cjs`: strict mode, module-level comments, clean exports.

   - Function returns `''` for non-string inputs
   - Module exports `{ escapeXml }`
   - No external dependencies

Create `bin/repowise/pack-file.cjs`:

1. `#!/usr/bin/env node` shebang and `'use strict'` directive
2. Module header: `// bin/repowise/pack-file.cjs — Pack file contents into XML <file> tags for LLM context delivery`
3. Require escapeXml from `./escape-xml.cjs`
4. Language detection map — `LANG_MAP` object mapping file extensions to language identifiers:
   ```
   const LANG_MAP = {
     '.js': 'js', '.mjs': 'js', '.cjs': 'js',
     '.ts': 'ts', '.tsx': 'tsx', '.jsx': 'jsx',
     '.py': 'py', '.rb': 'rb', '.go': 'go',
     '.rs': 'rs', '.java': 'java', '.kt': 'kotlin',
     '.c': 'c', '.cpp': 'cpp', '.h': 'c', '.hpp': 'cpp',
     '.cs': 'cs', '.php': 'php', '.swift': 'swift',
     '.sh': 'sh', '.bash': 'sh', '.zsh': 'sh',
     '.yml': 'yaml', '.yaml': 'yaml',
     '.json': 'json', '.xml': 'xml', '.html': 'html', '.css': 'css',
     '.md': 'markdown', '.sql': 'sql',
     '.tla': 'tla', '.als': 'alloy', '.pml': 'prism',
   };
   ```
5. `detectLang(filePath)` function:
   - Extracts extension from filePath using `path.extname()`
   - Returns `LANG_MAP[ext]` or `null` if unknown
6. `packFile({ filePath, content, lang })` function:
   - Parameters: `filePath` (string, required), `content` (string, required), `lang` (string, optional)
   - If `lang` is not provided, auto-detect from `filePath` via `detectLang()`
   - Escapes `content` using `escapeXml()`
   - Produces: `<file path="${escaped_path}" lang="${lang}">${escaped_content}</file>`
   - Note: `filePath` is NOT escaped — it's a path, not content; paths with XML-special chars are extremely rare and the attribute quoting handles most cases. If needed, a future version can add path escaping.
   - If `lang` resolves to `null` (unknown extension), omit the `lang` attribute entirely: `<file path="${path}">${content}</file>`
   - Returns the packed string (does not print to stdout — that's context-packer's job)
7. Exports: `module.exports = { packFile, detectLang, LANG_MAP };`

Follow the code style of existing bin/ modules (strict mode, clear comments, named exports).
