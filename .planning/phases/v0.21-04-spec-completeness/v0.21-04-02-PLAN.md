**Read both files fully before making any changes.**

**Audit walkthrough — compare these specific points:**

1. **Run-collapse algorithm (CollapseRuns action in TLA+ vs hooks/qgsd-circuit-breaker.js `detectOscillation`)**
   - JS: consecutive commit entries with the same `fileSets` key are merged into one run-group (lines that implement `fileSets.slice().sort().join('\0')`)
   - TLA+: `CollapseRuns` action produces run-length encoding — check if the label abstraction correctly models the JS string key behavior
   - Document: Does TLA+ CollapseRuns correctly model JS run-group counting? Y/N + explanation

2. **Depth threshold (Depth constant vs JS oscillation threshold)**
   - JS: what is the concrete threshold for minimum alternating groups before flagging?
   - TLA+: `Depth` constant, default 3 in MCoscillation.cfg
   - Document: Does Depth=3 match JS behavior? Y/N + line reference

3. **Second-pass net-diff logic (hasReversionInHashes) — CRITICAL**
   - JS: `hasReversionInHashes` iterates consecutive commit pairs, accumulates `(additions - deletions)` across all pairs, returns true if the net sum is <= 0
   - TLA+: `SetNetChange(n)` nondeterministically sets netChange; `EvaluateFlag` checks `netChange <= 0`
   - The nondeterministic abstraction is intentional but verify it captures all real behaviors
   - Document: Does the nondeterministic model correctly over-approximate the JS algorithm? Specifically: can TLC explore states where netChange > 0 (TDD progression) and netChange <= 0 (genuine reversion)?

4. **Flag condition (EvaluateFlag vs JS final decision)**
   - TLA+: `flagged' = (\E l \in Labels : flagCount[l] >= Depth /\ netChange <= 0)`
   - JS: flag only if BOTH run-count >= threshold AND net-diff <= 0 (hasReversionInHashes=true)
   - Document: Does `OscillationFlaggedCorrectly` invariant precisely encode the JS conjunction? Y/N

**If divergences found:**
- Update QGSDOscillation.tla to match the correct JS behavior
