<objective>
Build the core co-change prediction module: mine file co-occurrence from git history, compute temporal coupling scores with configurable thresholds, and filter mass-refactoring commits.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Create cochange.cjs (COCH-01, COCH-02, COCH-03)</name>
  <files>bin/repowise/cochange.cjs</files>
  <action>
Create `bin/repowise/cochange.cjs`:

1. `#!/usr/bin/env node` shebang, `'use strict'`
2. Module header: `// bin/repowise/cochange.cjs — Git-log co-change prediction for Repowise`
3. Imports: `fs`, `path`, `{ spawnSync }` from `child_process`, `{ parseGitNumstat, isExcluded, DEFAULT_EXCLUDE_PATTERNS, DEFAULT_MASS_REFACTOR_THRESHOLD }` from `./hotspot.cjs`, `{ escapeXml }` from `./escape-xml.cjs`

4. `computeCoChange(projectRoot, options)` — main API:
   - options: `{ minSharedCommits?, minCouplingDegree?, massRefactorThreshold?, excludePatterns?, since? }`
   - Parse git numstat via `parseGitNumstat(projectRoot, options)`
   - For each commit: generate all sorted file pairs from non-excluded files
   - Weight pairs: `1 / max(1, fileCount / threshold)` for mass-refactor commits
   - Accumulate pair counts and per-file commit counts
   - Compute coupling degree: `shared / min(commits_A, commits_B)`
   - Filter by minSharedCommits (default 3) and minCouplingDegree (default 0.3)
   - Return: `{ pairs: [{ file1, file2, shared_commits, coupling_degree }], summary: { total_pairs, strong_coupling_count }, fileIndex: Map<filePath, [partner entries]> }`

5. `formatCoChangeXml(cochange)` — formats as XML:
   ```xml
   <pairs>
     <pair file1="a.js" file2="b.ts" shared_commits="5" coupling_degree="0.45"/>
   </pairs>
   ```

6. `getPartnersForFile(filePath, cochange)` — returns array of co-change partners for a given file

7. CLI: `--project-root`, `--json`, `--min-shared-commits=N`, `--min-coupling-degree=F`, `--since=date`, `--help`

8. Exports: `{ computeCoChange, formatCoChangeXml, getPartnersForFile }`

9. `require.main === module` guard
  </action>
</task>
