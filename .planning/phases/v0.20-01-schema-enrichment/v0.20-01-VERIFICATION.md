## Key Implementation Details

### Schema Changes (Plan 01)

- Added 7 optional properties to `properties` object
- `runtime_ms` typed as `integer` with `minimum: 0` (critical for timeout thresholds)
- `observation_window` typed as `object` (PRISM-specific metadata)
- `triage_tags` typed as `array` of `string` (anomaly annotations)
- Required array unchanged for backwards compatibility

### Writer Implementation (Plan 02)

- **Validation:** Throws with descriptive error if any of 5 required fields missing
- **Field mapping:** Accepts `entry.check_id`, `entry.surface`, etc.; constructs `record` with same keys at top level
- **Math.round():** Applied to `runtime_ms` to ensure integer storage
- **Conditional inclusion:** `observation_window` only added if provided (not null)
- **Defaults:** `triage_tags` defaults to `[]`; `metadata` defaults to `{}`

### Caller Implementation (Plans 03-04)

- **Timer placement:** `const _startMs = Date.now()` immediately before `spawnSync`/`exec` (not at module load)
- **CHECK_ID_MAP:** Multi-config runners use map for distinct check IDs per configuration
- **Hardcoded IDs:** Single-config runners use hardcoded `check_id` string
- **Timeout annotations:** `triage_tags` logic per tool:
  - TLA+ (tla): `['timeout-risk']` when `runtime_ms > 120000ms`
  - Alloy: `['timeout-risk']` when `runtime_ms > 60000ms`
  - PRISM: `['timeout-risk']` (>300s), `['slow-verify']` (>120s), `['low-confidence']` (small window)
  - CI: `[]` (no timeout risk)
  - Liveness inconclusive: `['needs-fairness']` (run-oscillation-tlc)

### PRISM-Specific Changes

- **observation_window promotion:** Moved from `metadata` to top-level parameter in `writeCheckResult` call
- **Window structure:** Contains `window_start` (ISO timestamp), `window_end` (ISO timestamp), `n_traces` (quorum round count), `n_events` (total vote events), `window_days` (float days)
- **Low-confidence logic:** Automatic annotation when insufficient observation window
