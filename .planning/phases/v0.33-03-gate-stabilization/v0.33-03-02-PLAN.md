  <files>bin/compute-per-model-gates.cjs</files>
  <action>
Modify `bin/compute-per-model-gates.cjs` to integrate gate-stability.cjs:

1. **Import gate-stability module** at the top of the file (near other requires):
   ```javascript
   const { detectFlipFlops, isCooldownSatisfied, updateCooldownState, createUnstableEntry } = require('./gate-stability.cjs');
   ```

2. **Bump schema_version** in `writePerModelGateFile()` from `'2'` to `'3'` to reflect the new stability fields.

3. **Add `unstable_models` count** to the output object in `writePerModelGateFile()`, alongside `total_models` and `evidence_readiness`.

4. **Run flip-flop detection BEFORE the promotion loop** (Pitfall 2 from research -- detection must precede promotion decision):
   - After loading the changelog (around line 62-69 where changelog is read), call `detectFlipFlops(changelog)` to get the set of unstable models.
   - Load previous per-model-gates.json to read existing stability/cooldown state for each model.

5. **Gate the auto-promotion decision** (in the promotion loop at lines 446-488):
   - Before promoting a model, check: is this model in the unstable set OR does it have stability_status === 'UNSTABLE' from a previous run?
   - If yes, call `isCooldownSatisfied(modelStabilityInfo)`. If cooldown NOT satisfied, skip promotion and log to stderr: `[compute-per-model-gates] Promotion blocked for ${modelPath}: UNSTABLE, cooldown not satisfied (${sessions}/${requiredSessions} sessions, wall time ${elapsed}/${required}ms)`
   - If cooldown IS satisfied, allow promotion and reset stability_status to 'STABLE'.

6. **Update cooldown counters** after computing all gate scores for all models:
   - For each model flagged as UNSTABLE, call `updateCooldownState()` with the model's current evidence score (layer_maturity) and the promotion threshold (1).
   - Store the updated stability info in the per-model results.

7. **Add stability fields to each model entry** in the output:
   - `stability_status`: 'STABLE', 'UNSTABLE', or 'COOLDOWN' (COOLDOWN = was UNSTABLE, cooldown in progress)
   - `direction_changes`: number of alternations found
   - `cooldown`: null (if stable) or the cooldown object (if unstable/cooldown)

8. **Preserve existing behavior**: All existing fields, hysteresis thresholds, and promotion/demotion logic remain unchanged. The stability check is an additional gate BEFORE the existing promotion decision, not a replacement.

Important: Every blocked promotion MUST produce a stderr warning with `[compute-per-model-gates]` prefix (existing TAG convention from research anti-patterns).
