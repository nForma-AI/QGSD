<task type="auto">
  <name>Task 1: Create gate-stability.cjs module with flip-flop detection and cooldown logic</name>
  <files>bin/gate-stability.cjs</files>
  <action>
Create `bin/gate-stability.cjs` as a CommonJS module exporting the following functions:

1. **`countDirectionChanges(entries)`** -- Takes an array of changelog entries (each with `from_level` and `to_level` fields). Returns the number of direction alternations.
   - Use `LEVEL_ORDER = ['ADVISORY', 'SOFT_GATE', 'HARD_GATE']` to determine direction (up vs down).
   - Track `lastDirection` (null initially). For each entry, compute direction from level indices.
   - Skip entries where `from_level` or `to_level` is unknown or equal (same-level transitions).
   - Increment `changes` when direction differs from `lastDirection` (and lastDirection is not null).
   - Return the count.
   - IMPORTANT: Same-direction consecutive entries (e.g., two promotions in a row) do NOT count as alternations. Only actual direction reversals count. This prevents duplicate changelog entries from inflating the count (Pitfall 1 from research).

2. **`detectFlipFlops(changelog, threshold)`** -- Takes full changelog array and optional threshold (default 3). Returns object mapping model paths to `{ direction_changes, flagged_at }` for models exceeding threshold.
   - Group entries by `model` field.
   - For each model, call `countDirectionChanges()` on its entries.
   - Return only models where `direction_changes >= threshold`.

3. **`isCooldownSatisfied(stabilityInfo)`** -- Takes a model's stability object (from per-model-gates.json). Returns boolean.
   - If no stabilityInfo or stability_status !== 'UNSTABLE', return true (no cooldown needed).
   - If no cooldown object, return true.
   - Check BOTH conditions: `consecutive_stable_sessions >= required_sessions` AND `(Date.now() - flagged_at) >= required_wall_time_ms`.
   - Return true only if BOTH are met.

4. **`updateCooldownState(existingStability, currentGateScore, gateThreshold)`** -- Updates cooldown counters for a model.
   - If stability_status is not 'UNSTABLE', return existingStability unchanged.
   - If `currentGateScore >= gateThreshold` (model still passes), increment `consecutive_stable_sessions` and update `last_session_timestamp`.
   - If `currentGateScore < gateThreshold` (model regressed), reset `consecutive_stable_sessions` to 0 (Pitfall 3 from research).
   - Return updated stability object.

5. **`createUnstableEntry(directionChanges)`** -- Creates a fresh UNSTABLE stability object.
   - Returns: `{ stability_status: 'UNSTABLE', direction_changes, flagged_at: new Date().toISOString(), cooldown: { consecutive_stable_sessions: 0, required_sessions: 3, required_wall_time_ms: 3600000, last_session_timestamp: null } }`

6. **Constants**: Export `LEVEL_ORDER`, `DEFAULT_FLIP_FLOP_THRESHOLD = 3`, `DEFAULT_REQUIRED_SESSIONS = 3`, `DEFAULT_REQUIRED_WALL_TIME_MS = 3600000`.

Module exports all functions and constants. Use `module.exports = { ... }`.
