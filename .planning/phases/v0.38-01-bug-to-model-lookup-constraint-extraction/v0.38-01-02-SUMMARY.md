- **Found during:** Task 2 (test execution)
- **Issue:** VARIABLES regex matched `vars == <<...>>` on the next line as part of the variable list, causing "disabled" and "valid" to be missed
- **Fix:** Simplified regex to single-line capture `/VARIABLES\s+([^\n]+)/` and filter valid identifiers
- **Files modified:** bin/model-constrained-fix.cjs
- **Verification:** All 30 tests pass
- **Committed in:** b2158a84 (Task 2 commit)

**2. [Rule 1 - Bug] Fixed TLA+ definition body extraction bleeding across boundaries**
- **Found during:** Task 1 (verification)
- **Issue:** Regex-based extraction bled definition bodies across comment blocks and into subsequent definitions
- **Fix:** Replaced single-regex approach with position-based extraction using definition positions, section markers, and blank line boundaries
- **Files modified:** bin/model-constrained-fix.cjs
- **Verification:** Clean output confirmed for both NFCircuitBreaker.tla and NFConfigLoader.tla
- **Committed in:** bdae0527 (Task 1 commit, iterative fix before final commit)
