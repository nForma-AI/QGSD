# Phase v0.9-01 Plan 01: Context Window Monitor Hook Summary

PostToolUse hook (`gsd-context-monitor.js`) that injects WARNING at >= 70% context used and CRITICAL at >= 90% used, with thresholds overridable via `qgsd.json` `context_monitor` block.

## What Was Built

`hooks/gsd-context-monitor.js` is a stateless PostToolUse hook. It reads `context_window.remaining_percentage` from the hook event input, computes `usedPct = Math.round(100 - remaining)`, and injects one of two messages into `additionalContext`:

- **WARNING** (`usedPct >= warn_pct`, default 70): instructs the model to wrap up and avoid starting new complex work
- **CRITICAL** (`usedPct >= critical_pct`, default 90): instructs the model to stop new work immediately and save state via `/qgsd:pause-work`
- **No output** (below warn_pct): `process.exit(0)` silently

Config is loaded via `loadConfig()` from the shared two-layer config-loader. Both thresholds default to values in `DEFAULT_CONFIG.context_monitor` and can be overridden in project or global `qgsd.json`.
