  <action>
Create `bin/accept-debug-invariant.cjs` using only Node.js built-in `fs` and `path` modules.

**CLI interface:**
```
node bin/accept-debug-invariant.cjs <target-spec.tla> <property-name> <property-body> --session-id <id>
```
- `<target-spec.tla>`: Absolute or relative path to canonical spec (e.g., `formal/tla/QGSDQuorum.tla`)
- `<property-name>`: TLA+ property name (e.g., `DebugSessionInvariant`)
- `<property-body>`: TLA+ formula body as a string (e.g., `"x > 0 /\ y < MaxSize"`)
- `--session-id <id>`: Required. Format: `debug-sess-<unix-timestamp-seconds>-<8-char-hex>`. Fail if missing.

**Algorithm (from RESEARCH.md Pattern 3):**
1. Parse CLI arguments. Validate session_id is present — if missing: `process.stderr.write('Error: --session-id is required for debug invariant writes\n'); process.exit(1);`
2. Read target spec content. If file missing: `process.stderr.write('Error: spec not found: ' + targetPath + '\n'); process.exit(1);`
3. Check if property name already exists in target:
   - Use regex: `new RegExp('^PROPERTY\\s+' + propertyName + '\\b', 'm').test(specContent)`
   - If match: `process.stderr.write('Error: PROPERTY ' + propertyName + ' already exists in spec\n'); process.exit(1);`
4. Build the invariant block to append:
   ```
   \n(* DEBUG SESSION: <session-id> *)\nPROPERTY <name> == <body>\n
   ```
5. Merge: if spec ends with `====` (TLA+ end marker), insert before it; otherwise append.
6. Write spec atomically via tmp+rename:
   ```js
   const ROOT = path.join(__dirname, '..');
   const tmpSpec = targetPath + '.tmp.' + Date.now() + '.' + Math.random().toString(36).slice(2);
   fs.writeFileSync(tmpSpec, mergedContent, 'utf8');
   fs.renameSync(tmpSpec, path.resolve(targetPath));
   ```
7. Update model-registry.json atomically:
   - Read registry from `path.join(ROOT, 'formal', 'model-registry.json')`. If missing: warn to stderr, do not fail (fail-open).
   - Compute registry key: `path.relative(ROOT, path.resolve(targetPath)).replace(/\\/g, '/')`.
   - Existing entry or empty: update fields:
     ```js
     registry.models[key] = {
       version: (existing.version || 0) + 1,
       last_updated: new Date().toISOString(),
       update_source: 'debug',
       source_id: null,
       session_id: sessionId,   // from --session-id flag
       description: existing.description || ''
     };
     registry.last_sync = new Date().toISOString();
     ```
   - Write via tmp+rename.
8. Print success: `[accept-debug-invariant] Wrote PROPERTY <name> to <target>. session_id: <id>. Registry version: <new version>`.

**Session ID generation for callers** (include as comment at bottom of file, not executable):
```
// Generate session ID: `debug-sess-${Math.floor(Date.now()/1000)}-${Math.random().toString(16).slice(2,10)}`
```
This helps the /qgsd:debug orchestrator know the expected format when calling this script.
