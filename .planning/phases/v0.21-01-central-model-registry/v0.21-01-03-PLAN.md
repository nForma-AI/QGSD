  <action>
Create `bin/promote-model.cjs` using only Node.js built-in `fs` and `path` modules.

**CLI interface:**
```
node bin/promote-model.cjs <proposed-changes.tla> <target-spec.tla> [--source-id <id>]
```
- `<proposed-changes.tla>`: Path to proposed spec (e.g., `.planning/phases/v0.21-01-central-model-registry/formal/proposed-changes.tla`)
- `<target-spec.tla>`: Path to canonical target (e.g., `formal/tla/QGSDQuorum.tla`)
- `--source-id <id>`: Optional provenance ID (e.g., `v0.21-01-003`); defaults to `null`

**Algorithm (from RESEARCH.md Pattern 1):**
1. Read proposed-changes.tla content.
2. Extract PROPERTY definitions using regex: `/^PROPERTY\s+(\w+)\s*==\s*([\s\S]*?)(?=\nPROPERTY\s|\nEND\s|\n\\\*/|\Z)/gm`
   - Simpler alternative: split on `\nPROPERTY ` boundaries and parse name from each fragment.
   - Extract just the property names: `/^PROPERTY\s+(\w+)/gm` across proposed content.
3. Read target spec content.
4. Extract existing PROPERTY names from target using same regex.
5. Validate no name conflicts: find intersection of proposed names and existing names.
   - If intersection non-empty: `process.stderr.write('Error: duplicate PROPERTY names: ' + [...intersection].join(', ') + '\n'); process.exit(1);`
6. Merge: append proposed PROPERTY blocks to the end of the target content.
   - Find the last line of the target spec (typically `=====` or similar end marker for TLA+).
   - If target ends with `====` (TLA+ module end), insert proposed content BEFORE the `====` line.
   - Otherwise append at end.
   - Add a blank line separator between existing content and proposed additions.
7. Write merged spec atomically:
   ```js
   const ROOT = path.join(__dirname, '..');
   const tmpSpec = targetPath + '.tmp.' + Date.now() + '.' + Math.random().toString(36).slice(2);
   fs.writeFileSync(tmpSpec, mergedContent, 'utf8');
   fs.renameSync(tmpSpec, targetPath);
   ```
8. Update model-registry.json atomically:
   - Read existing registry from `path.join(ROOT, 'formal', 'model-registry.json')`.
   - Compute registry key: `path.relative(ROOT, path.resolve(targetPath)).replace(/\\/g, '/')`.
   - Increment version, set last_updated=now, update_source="plan-promote", source_id=from CLI arg or null.
   - Write via tmp+rename (same pattern as spec write).
9. Print success: `[promote-model] Merged N properties into <target>. Registry version: <new version>`.

**Error handling:**
- Proposed file not found → stderr + exit(1)
- Target file not found → stderr + exit(1)
- Registry not found → warn to stderr but do NOT fail (fail-open on registry)
- Duplicate property names → stderr + exit(1)

**Note on TLA+ PROPERTY extraction:** TLA+ PROPERTY syntax is `PROPERTY <name> == <formula>`. The formula can span multiple lines until the next `PROPERTY`, `INVARIANT`, `=====`, or EOF. Use a simple approach: find all lines starting with `PROPERTY ` and extract the word after it as the name. This is sufficient for conflict detection.
