  <action>
Create bin/diagnostic-proposal-generator.cjs that exports `generateCorrectionProposals(stateDiff, bugContext)`.

Input: `stateDiff` is the output of `generateStateDiff()` from diagnostic-diff-generator.cjs (Phase 1). It has shape: `{ summary: { changed_fields, first_divergence_index, total_changes }, per_state_diffs: [{index, changes: [{key, oldValue, newValue}]}] }`. `bugContext` is a string (the bug description text).

Output: Array of proposal objects sorted by priority (ascending), each with:
```
{
  id: string,           // e.g., "PROP-SV-0", "PROP-INV-1", "PROP-TEMP-1"
  type: string,         // "add_state_variable" | "add_invariant" | "add_temporal_property"
  target: string,       // The specific element to add (field name, invariant name, property name)
  confidence: number,   // 0.0–1.0
  reasoning: string,    // Human-readable explanation citing CONCRETE EVIDENCE from trace alignment
  example: string,      // TLA+/Alloy skeleton example showing what to add
  priority: number,     // 1=highest (state variables), 2=medium (invariants), 3=lower (temporal)
  effort: string        // "low" | "medium" | "high"
}
```

**CRITICAL — Evidence-based reasoning (not field-name heuristics):**

Proposals MUST derive their reasoning from the actual divergence evidence in `per_state_diffs[].changes[]`, not from field names alone. The `changes` array contains `{ key, oldValue, newValue }` tuples where `oldValue` is the model's value and `newValue` is the bug trace's value. Use these concrete values in reasoning and examples.

Three proposal categories (generate all that apply):

1. **add_state_variable** (priority 1, effort "low") — One per diverged field in `summary.changed_fields`. For EACH field, look up its first change entry in `per_state_diffs` to extract `oldValue` (model) and `newValue` (bug trace).
   - ID: `PROP-SV-{i}`.
   - Target: the field name.
   - Confidence: 0.9 if field appears in bugContext string (case-insensitive match), 0.7 if not mentioned in bug context (model-only divergence).
   - Reasoning: MUST cite concrete values: "Field '{field}' diverges: model produces {oldValue} but bug trace shows {newValue} at state {index}. Model does not track this variable correctly." If the field name appears in bugContext, append: "Field is explicitly mentioned in the bug description, confirming relevance."
   - Example: `VARIABLE {field} \\in {inferred_type_from_values} \\\\ Bug expects {newValue}, model produces {oldValue}` where `inferred_type_from_values` is `Nat` if both values are numbers, `BOOLEAN` if both are booleans, `STRING` if strings, `{oldValue, newValue}` as an explicit set otherwise.

2. **add_invariant** (priority 2, effort "medium") — One proposal if `per_state_diffs` is non-empty AND `first_divergence_index` is not null. Extract the FIRST per_state_diff entry's changes to build concrete invariant.
   - ID: `PROP-INV-1`.
   - Target: `Inv_StateConstraint_{index}`.
   - Confidence: 0.8 if the divergence has 1-2 changed fields at that state (focused), 0.6 if 3+ changed fields (diffuse divergence, harder to constrain).
   - Reasoning: MUST cite evidence: "Divergence first occurs at state {index} with {N} field(s) changing: {list first 3 field:oldValue->newValue}. Add invariant constraining valid transitions at this state." Do NOT use generic "add invariant constraining valid state transitions".
   - Example: `Inv_{index} == pc = {index} => {field1} {op} {newValue1} /\\ {field2} {op} {newValue2}` using actual field names and values from changes. Use `=` for exact match, `\\in` for range if values are numeric.

3. **add_temporal_property** (priority 3, effort "high") — One proposal if `changed_fields` is non-empty. Use the first changed field's divergence values.
   - ID: `PROP-TEMP-1`.
   - Target: `FinalStateCorrectness`.
   - Confidence: 0.6.
   - Reasoning: "Model's final state has {field}={oldValue} but bug trace ends with {field}={newValue}. Add temporal property ensuring model eventually reaches the correct terminal state."
   - Example: `Prop_FinalCorrect == <>[](${field} = ${newValue})` using actual values from the divergence.

Return empty array if stateDiff has no changes (`per_state_diffs` empty and `changed_fields` empty).

Sort output by `priority` ascending (1 before 2 before 3).

Create bin/diagnostic-proposal-generator.test.cjs with 10+ tests using `require('node:test')` and `assert`:
