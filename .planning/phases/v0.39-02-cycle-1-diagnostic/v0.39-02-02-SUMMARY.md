# Phase v0.39-02 Plan 02: Evidence-Based Language Summary

**Reframe all user-facing output from "bug not captured" to "model is incomplete/wrong" across refinement loop and model-driven-fix workflows.**

## Overview

Updated human-facing output strings across refinement-loop.cjs and workflow markdown files (model-driven-fix.md, close-formal-gaps.md) to use evidence-based language reflecting the model-as-hypothesis paradigm. The bug is ground truth (oracle); the model is the hypothesis under test. Current language ("bug not captured") implies the bug description might be wrong. Corrected language ("model is incomplete") guides users toward improving the model, not re-describing the bug.
