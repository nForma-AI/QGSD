### Task 2: Diagnostic Proposal Generator (16 tests pass)

Created `bin/diagnostic-proposal-generator.cjs` exporting:

- **`generateCorrectionProposals(stateDiff, bugContext)`** — Generates structured correction proposals from state divergence

**Input contract:**
```javascript
stateDiff: {
  summary: { changed_fields: string[], first_divergence_index: number|null },
  per_state_diffs: [{ index, changes: [{ key, oldValue, newValue }] }]
}
bugContext: string (bug description for context-aware scoring)
```

**Output:** Array of proposal objects (sorted by priority ascending) with shape:
```javascript
{
  id: "PROP-SV-0" | "PROP-INV-1" | "PROP-TEMP-1",
  type: "add_state_variable" | "add_invariant" | "add_temporal_property",
  target: string,
  confidence: number (0.0–1.0),
  reasoning: string (evidence-based, cites concrete values),
  example: string (TLA+ skeleton with actual values),
  priority: number (1=highest, 2=medium, 3=lower),
  effort: "low" | "medium" | "high"
}
```

**Three proposal categories (all evidence-based):**

1. **add_state_variable** (priority 1, effort "low") — One per diverged field
   - ID: `PROP-SV-{i}`
   - Target: field name
   - **Confidence: context-aware**
     - 0.9 if field name appears in bugContext (case-insensitive match)
     - 0.7 if field not mentioned in bugContext (model-only divergence)
   - **Reasoning: cites concrete values**
     - Format: "Field 'X' diverges: model produces {oldValue} but bug trace shows {newValue} at state {index}"
     - If field in bugContext, appends: "Field is explicitly mentioned in the bug description"
   - **Example: uses inferred type**
     - Type inference: Nat for numbers, BOOLEAN for bools, STRING for strings, explicit set otherwise
     - Format: `VARIABLE X \in {type} \\ Bug expects {newValue}, model produces {oldValue}`

2. **add_invariant** (priority 2, effort "medium") — One if first_divergence_index is non-null
   - ID: `PROP-INV-1`
   - Target: `Inv_StateConstraint_{index}`
   - **Confidence: focused vs. diffuse divergence**
     - 0.8 for 1-2 changed fields (focused, clear constraint)
     - 0.6 for 3+ changed fields (diffuse, harder to constrain)
   - **Reasoning: lists concrete field:oldValue->newValue pairs**
     - Format: "Divergence first occurs at state {index} with {N} field(s) changing: {field1}:{oldValue1}->{newValue1}, {field2}:..."
   - **Example: uses actual field names and values**
     - Format: `Inv_{index} == pc = {index} => {field1} = {newValue1} /\ {field2} = {newValue2}`

3. **add_temporal_property** (priority 3, effort "high") — One if changed_fields is non-empty
   - ID: `PROP-TEMP-1`
   - Target: `FinalStateCorrectness`
   - Confidence: 0.6 (temporal properties require deep understanding)
   - **Reasoning: cites final state divergence**
     - Format: "Model's final state has {field}={oldValue} but bug trace ends with {field}={newValue}"
   - **Example: uses actual terminal values**
     - Format: `Prop_FinalCorrect == <>[]({field} = {newValue})`

**Evidence-based reasoning (not field-name heuristics):**
Every proposal reasoning cites concrete `oldValue`/`newValue` pairs extracted from `per_state_diffs[].changes[]`, never just field names. Examples show actual trace values to guide developer corrections.

## Evidence-Based Reasoning Implementation

The diagnostic proposal generator implements **evidence-based reasoning** as a core requirement:

### Before (Generic Approach)
```javascript
// Bad: Generic reasoning without evidence
reasoning: "Model does not track this variable"
```

### After (Evidence-Based Approach)
```javascript
// Good: Evidence-based reasoning with concrete values
reasoning: "Field 'count' diverges: model produces 3 but bug trace shows 10 at state 1. " +
           "Field is explicitly mentioned in the bug description, confirming relevance."
```

Each proposal includes:
- **Concrete observed values**: "model produces 3 but bug trace shows 10" (not "model wrong")
- **State location**: "at state 1" pinpoints divergence location
- **Context relevance**: "Field is explicitly mentioned" provides confidence signal
- **Type inference**: Actual types (Nat, BOOLEAN) deduced from values

This enables quorum workers to reason about *specific* missing elements rather than guessing at improvements.
