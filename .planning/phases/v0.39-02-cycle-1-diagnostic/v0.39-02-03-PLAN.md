**New internal function `generateDiagnosticFeedback(checkerOutput, bugTraceJsonPath, bugContext)`:**
- Takes checker stdout/stderr output, path to bug trace ITF JSON, and bug context string.
- Stage 1: Write checker output to a temp ITF file (in /tmp) if it looks like JSON, then call `extractFinalStates(tempPath)` to get model's final states. If checker output is not parseable JSON, return null (fail-open).
- Stage 2: Call `extractFinalStates(bugTraceJsonPath)` to get bug trace's final states. If bugTraceJsonPath is null/undefined or file doesn't exist, return null (fail-open — diagnostic requires bug trace).
- Stage 3: Call `generateStateDiff(modelFinalStates, bugFinalStates)` to compare.
- Stage 4: Call `formatDiffAsMarkdown(diffResult)` for human-readable output.
- Stage 5: Call `generateCorrectionProposals(diffResult, bugContext)` for structured proposals.
- Return `{ mismatch_diff: markdownString, correction_proposals: proposalArray, trace_alignment: { model_state_count, bug_state_count, first_divergence_index, diverged_fields } }`.
- Wrap entire function in try/catch; on error return null with stderr warning. This is fail-open — diagnostic failure must NOT break the refinement loop.

**Update `verifyBugReproduction` options:**
- Add new optional parameters: `options.bugTraceJsonPath` (string, path to bug trace ITF JSON) and `options.onDiagnosticGenerated` (callback function receiving `(diagnostic, attemptNumber)`).

**Update the INCOMPLETE branch inside the iteration loop (around line ~192):**
After recording `iteration = { attempt, passed: true, summary }`:
```javascript

if (options.bugTraceJsonPath && attempt < maxAttempts) {
  const diagnostic = generateDiagnosticFeedback(output, options.bugTraceJsonPath, bugContext);
  if (diagnostic) {
    iteration.diagnostic = diagnostic;
    if (options.onDiagnosticGenerated) {
      options.onDiagnosticGenerated(diagnostic, attempt);
    }
  }
}
```

**Sub-change A: Update quorum-slot-dispatch.cjs to format diagnostic review-context**

The existing `--review-context` flag in quorum-slot-dispatch.cjs already passes a string into `buildModeAPrompt` and `buildModeBPrompt` via the `reviewContext` parameter. Update the prompt builders to detect and format diagnostic JSON within review-context:

1. In `buildModeAPrompt`, before the existing reviewContext handling, add a diagnostic context formatter:
```javascript
// If reviewContext contains diagnostic JSON (from refinement loop), format it as a structured section
let diagnosticSection = '';
if (reviewContext) {
  try {
    const parsed = JSON.parse(reviewContext);
    if (parsed.mismatch_diff && parsed.correction_proposals) {
      diagnosticSection = '\n\n## Model Diagnostic Feedback\n\n' +
        'The model is INCOMPLETE. It does not capture the bug. ' +
        'Here is what the model assumes vs. what the bug trace shows:\n\n' +
        parsed.mismatch_diff + '\n\n' +
        '### Correction Proposals (ranked by priority)\n\n' +
        (parsed.correction_proposals || []).map((p, i) =>
          `${i + 1}. **[${p.type}]** ${p.target}: ${p.reasoning}\n   Example: \`${p.example}\``
        ).join('\n') + '\n\n' +
        '### Trace Alignment\n' +
        `- Model states: ${parsed.trace_alignment?.model_state_count || '?'}\n` +
        `- Bug trace states: ${parsed.trace_alignment?.bug_state_count || '?'}\n` +
        `- First divergence: state ${parsed.trace_alignment?.first_divergence_index ?? '?'}\n` +
        `- Diverged fields: ${(parsed.trace_alignment?.diverged_fields || []).join(', ')}\n\n` +
        'Use these proposals to guide your model refinement. Address high-priority proposals first.';
      // Replace reviewContext with formatted section so it doesn't double-render
      reviewContext = null;
    }
  } catch (_) {
    // Not JSON — leave reviewContext as-is for normal string handling
  }
}
```

2. Insert `diagnosticSection` into the prompt string after the question/artifact section but before the response format instructions. This ensures diagnostic context is visible to the worker but doesn't interfere with the voting format.
