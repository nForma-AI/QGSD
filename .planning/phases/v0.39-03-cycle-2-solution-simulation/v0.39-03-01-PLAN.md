  <action>
Create bin/intent-normalizer.cjs implementing normalizeFixIntent(intentText, context).

The module accepts fix ideas in three input channels and normalizes them to a structured mutation array:

1. **Channel 1 — Constraint syntax:** Parse patterns like "if X then Y", "when X, Y must hold", "X unless Y" into add_invariant mutations. Use regex: /(?:if|when)\s+([^,]+?)(?:then|,?)\s+([^.;]+?)(?:\.|;|$)/gi

2. **Channel 2 — Variable syntax:** Parse patterns like "add timeout variable", "track retryCount", "set field X" into add_state_variable mutations. Use regex: /(?:add|set|track|initialize)\s+(?:state\s+)?(?:variable|field)?\s*(\w+)/gi. Deduplicate by lowercase name.

3. **Channel 3 — Code snippets:** Parse ```code blocks``` into code_modification mutations. Use regex: /```([^`]+?)```/g

Each mutation is an object: { type: string, target: string, content: string, reasoning: string }

Also compute:
- confidence: min(1.0, mutations.length / 3) or 0 if no mutations
- ambiguities: array of strings when hedging language detected ("maybe", "might", "could", "possibly", "perhaps") or multiple options ("or else", "or alternatively")

The context parameter (optional) provides: { bugDescription, reproducingModelPath, affectedCode }. When provided, include bugDescription in mutation reasoning strings for traceability.

Export: { normalizeFixIntent }
Module pattern: CommonJS (module.exports), consistent with all bin/*.cjs files.

Create bin/intent-normalizer.test.cjs with node:test and node:assert:
- Channel 1: constraint "if timeout then retry" produces add_invariant mutation
- Channel 1: multiple constraints parsed from compound sentence
- Channel 2: "add timeout variable" produces add_state_variable mutation
- Channel 2: duplicate variable names deduplicated
- Channel 3: code block produces code_modification mutation
- Mixed input: all three channels extract their respective mutations
- Ambiguity: hedging language detected
- Ambiguity: "or alternatively" detected
- Confidence: 0 mutations = confidence 0
- Confidence: 3+ mutations = confidence 1.0
- Empty input: returns empty mutations array, confidence 0
- Context parameter: bugDescription included in mutation reasoning when provided

<task type="auto">
  <name>Task 2: Consequence Model Generator Module</name>
  <files>bin/consequence-model-generator.cjs, bin/consequence-model-generator.test.cjs</files>
  <action>
Create bin/consequence-model-generator.cjs implementing generateConsequenceModel and applyMutation.

**generateConsequenceModel(reproducingModelPath, mutations, options = {})**
- options: { sessionId (default: crypto.randomBytes(8).toString('hex')), formalism: 'tla'|'alloy' (auto-detect from file extension) }
- Step 1: Read reproducing model file content
- Step 2: Auto-detect formalism from extension (.tla -> 'tla', .als -> 'alloy') unless overridden
- Step 3: Apply each mutation sequentially via applyMutation()
- Step 4: Create session directory at .planning/formal/cycle2-simulations/{sessionId}/
- Step 5: Write consequence model to consequence-model.{tla|als} in session directory
- Step 6: Write normalized-mutations.json to session directory (array of { iteration: 1, type, target, content, applied: true/false })
- Step 7: Return { consequenceModelPath, appliedMutations, sessionDir, diagnostics: { totalMutations, appliedCount, skippedCount } }

**applyMutation(modelContent, mutation, formalism)**
- For 'tla' formalism:
  - add_invariant: Append to existing Invariant section or create one. Pattern: find last line before "====" or end of spec, insert "Inv_UserMutation_{target} == {content}" before it
  - add_state_variable: Find VARIABLES declaration block, append new variable. Pattern: find /VARIABLES?\s/ line, append ", {target}" or add new VARIABLE line
  - code_modification: Append as comment block with content at end of spec before terminal "===="
- For 'alloy' formalism:
  - add_invariant: Append "fact {target} { {content} }" before closing brace or at end
  - add_state_variable: Append "one sig {target} extends State {}" or add field to existing sig
  - code_modification: Append as comment block
- Return { success: boolean, content: string, error: string|null }
- Fail-open: if mutation cannot be applied (parse error), set applied=false in record but continue with other mutations

Export: { generateConsequenceModel, applyMutation }
Module pattern: CommonJS.

- generateConsequenceModel: auto-detects formalism from .tla extension
- generateConsequenceModel: auto-detects formalism from .als extension

Use tmp directories for test isolation (fs.mkdtempSync). Clean up after each test.
