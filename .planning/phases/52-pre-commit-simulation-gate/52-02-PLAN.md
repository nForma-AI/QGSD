```javascript
onTweakFix: async (fixIdea, iterationContext) => {
  // GATE-05: Executor refines approach when convergence gates fail
  // iterationContext contains: { iteration, verdict, gatesPassing, bestGatesPassing, stuck_reason }

  const failedGates = [];
  if (iterationContext.verdict.gate1_invariants?.passed !== true) failedGates.push('invariants');
  if (iterationContext.verdict.gate2_bug_resolved?.passed !== true) failedGates.push('bug_resolved');
  if (iterationContext.verdict.gate3_neighbors?.passed !== true) failedGates.push('neighbors');

  if (failedGates.length === 0) return null; // All gates passed, stop

  // If stuck (same failure 3+ times), return null to let escalation handle it
  if (iterationContext.stuck_reason) return null;

  // Build refined fix description based on which gates failed
  const refinement = `Iteration ${iterationContext.iteration}: Gates failing: ${failedGates.join(', ')}. ` +
    `Previous fix idea: "${fixIdea}". ` +
    `Refine the approach to address the failing gate(s). ` +
    `Make ONE targeted edit to the changed files that preserves the intent while satisfying the formal model constraints.`;

  return refinement;
}
```

This callback:
1. Reads which gates failed from iterationContext.verdict
2. Returns null (stop) if all gates pass or if stuck (same failure pattern 3+ consecutive times)
3. Returns a refined fix description that tells the executor which gates failed and to make ONE targeted edit
4. The simulateSolutionLoop uses this returned description as the next iteration's fixIdea
