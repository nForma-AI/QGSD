  <name>Task 1: Wire LRU cache and session metrics into coderlm-adapter.cjs</name>
  <files>bin/coderlm-adapter.cjs</files>
  <action>
    Modify bin/coderlm-adapter.cjs to add LRU cache and session metrics.

    STEP A — Add cache + metrics to createAdapter():

    At the top of the file (after existing requires), add:
    ```
    const { createLRUCache } = require('./coderlm-cache.cjs');
    ```

    In createAdapter(opts), after building the adapter config, create:
    ```
    const _cache = createLRUCache(100, 5 * 60 * 1000);  // 100 entries, 5min TTL (CADP-01)
    const _metrics = { queryCount: 0, totalLatencyMs: 0 };
    ```

    STEP B — Cache all async query methods (getCallers, getImplementation, findTests, peek):

    For each async query method, wrap the implementation in a cache-check + metrics pattern:

    ```
    async getCallers(symbol, file) {
      if (!this.enabled) return { error: 'disabled' };
      const cacheKey = JSON.stringify({ method: 'getCallers', symbol, file });
      const cached = _cache.get(cacheKey);
      if (cached !== undefined) return cached;  // cache hit (counted by _cache stats)
      const start = Date.now();
      try {
        const result = await /* existing HTTP call */;
        _metrics.queryCount++;
        _metrics.totalLatencyMs += Date.now() - start;
        _cache.set(cacheKey, result);
        return result;
      } catch (e) {
        return { error: e.message };  // CADP-02: never throw
      }
    }
    ```

    Apply same pattern to getImplementation, findTests, peek.

    STEP C — Cache and instrument sync methods (getCallersSync, getImplementationSync):

    IMPORTANT: nf-solve.cjs uses the SYNC path (getCallersSync, getImplementationSync) for all solve loop queries. These MUST also track _metrics.queryCount and _metrics.totalLatencyMs, otherwise CADP-03 session metrics will show queryCount:0 even after many queries.

    For each sync method that makes a real coderlm query:
    ```
    getCallersSync(symbol, file) {
      if (!enabled) return { error: 'disabled' };
      const cacheKey = JSON.stringify({ method: 'getCallersSync', symbol, file });
      const cached = _cache.get(cacheKey);
      if (cached !== undefined) return cached;  // cache hit
      const start = Date.now();
      try {
        const result = /* existing spawnSync call */;
        _metrics.queryCount++;
        _metrics.totalLatencyMs += Date.now() - start;
        _cache.set(cacheKey, result);
        return result;
      } catch (e) {
        return { error: e.message };  // CADP-02: never throw
      }
    }
    ```

    Apply same pattern to getImplementationSync (added in Plan 02).

    healthSync does NOT need caching or metrics tracking (health checks should always be fresh and are not queries).

    STEP D — Add getSessionMetrics() and resetCache() to adapter object:

    ```
    getSessionMetrics() {
      const cacheStats = _cache.stats();
      return {
        queryCount: _metrics.queryCount,
        cacheHits: cacheStats.hits,
        cacheMisses: cacheStats.misses,
        cacheHitRate: cacheStats.hitRate,
        totalLatencyMs: _metrics.totalLatencyMs,
        avgLatencyMs: _metrics.queryCount > 0 ? _metrics.totalLatencyMs / _metrics.queryCount : 0,
      };
    },
    resetCache() {
      _cache.reset();
      _metrics.queryCount = 0;
      _metrics.totalLatencyMs = 0;
    }
    ```

    STEP E — CADP-02 call-site hardening: ensure every existing method has explicit try/catch returning { error: string } on exception. Review all methods and wrap any that currently rely on the outer pattern only. Sync methods: if spawnSync itself throws (e.g., ENOMEM), catch and return { error }.

    The cache and metrics variables are closed over by the adapter object — they are per-adapter-instance, not module-level globals. This means each createAdapter() call starts with a fresh cache and metrics, consistent with "per solve session" semantics (nf-solve creates one adapter per run).
