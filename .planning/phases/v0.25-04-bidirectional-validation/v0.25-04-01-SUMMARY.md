# Plan v0.25-04-01: Bidirectional Validation in Matrix Generator — Summary

## What was built

Added `validateBidirectionalLinks()` function to `bin/generate-traceability-matrix.cjs` that cross-validates model-registry `requirements` arrays against requirements.json `formal_models` arrays. The validation detects:

1. **Asymmetric links (direction: model_claims_requirement):** Model-registry lists a requirement for a model file, but requirements.json does not list that model in its `formal_models`.
2. **Asymmetric links (direction: requirement_claims_model):** Requirements.json lists a model file for a requirement, but model-registry does not list that requirement for the model.
3. **Stale links (unknown_requirement_id):** Model-registry references a requirement ID that does not exist in requirements.json.
4. **Stale links (missing_model_file):** Requirements.json references a model file that does not exist on disk.
