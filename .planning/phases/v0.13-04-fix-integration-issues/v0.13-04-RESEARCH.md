## Summary

This phase consists of two targeted text edits in QGSD workflow markdown files. Both issues were surfaced by the v0.13 milestone audit (MILESTONE-AUDIT.md), which classified them as integration issues of medium severity.

INT-01 is a binary path inconsistency: the three quorum-gate workflow files (`plan-milestone-gaps.md`, `execute-phase.md`, `discuss-phase.md`) invoke `update-scoreboard.cjs` via `$HOME/.claude/qgsd-bin/update-scoreboard.cjs`. Other binaries in the same files use `~/.claude/qgsd/bin/` (gsd-tools.cjs). Both paths exist on the current installation. The installer copies `bin/*.cjs` scripts to `qgsd-bin/` — confirmed in `bin/install.js` line 1729-1740. The `qgsd-bin` path is correct and canonical for `update-scoreboard.cjs`. The inconsistency is that the audit noted the two path patterns differ, but investigation confirms both paths are populated by the installer. The INT-01 fix is therefore a style-consistency alignment: confirm (or if truly inconsistent, align) all three calls to use one canonical path pattern.

INT-02 is residual user-gate text in `plan-milestone-gaps.md` Step 5. Three locations retain interactive prompting language that is not guarded by an `--auto` bypass: (1) line 107 — `Create these {X} phases? (yes / adjust / defer all optional)` inside a markdown code block, (2) line 36 — the `nice` priority table row reads `Ask user: include or defer?` with no auto-mode conditional, and (3) the `<success_criteria>` block line 318 — `User confirmed phase plan` which is stale now that quorum replaced the user gate. A general-purpose subagent running in autonomous mode may interpret these as instructions to pause and await user input.

**Primary recommendation:** Two targeted text edits — INT-01: align update-scoreboard.cjs path across three files to one canonical form; INT-02: remove/guard three user-gate text locations in plan-milestone-gaps.md.

## Architecture Patterns

### Pattern 1: Auto-mode bypass guard

**What:** When a workflow is invoked in autonomous mode (`--auto` flag or `workflow.auto_advance=true` config), interactive user prompts must be either removed or wrapped in a guard that skips them.

**When to use:** Anywhere a workflow file contains language like "ask user", "Create X? (yes/no)", or "User confirmed" when the workflow can also run in headless/autonomous mode.

**Example from execute-phase.md (HIGH confidence — direct codebase inspection):**

```markdown
**Auto-advance detection:**

1. Parse `--auto` flag from $ARGUMENTS
2. Read `workflow.auto_advance` from config:
   ```bash
   AUTO_CFG=$(node ~/.claude/qgsd/bin/gsd-tools.cjs config-get workflow.auto_advance 2>/dev/null || echo "true")
   ```

**If `--auto` flag present OR `AUTO_CFG` is true ...**
```

The pattern for plan-milestone-gaps.md INT-02 follows the same principle: the quorum approval gate (already in Step 5) IS the replacement for the user dialog. The three user-gate text locations should be removed or replaced with automation-compatible language.

### Pattern 2: Canonical path consistency

**What:** QGSD installs two separate binary directories:
- `~/.claude/qgsd-bin/` — receives `bin/*.cjs` files (update-scoreboard.cjs, call-quorum-slot.cjs, etc.)
- `~/.claude/qgsd/bin/` — receives `qgsd-core/bin/*.cjs` files (gsd-tools.cjs, gsd-tools.test.cjs)

**When to use:** Use `$HOME/.claude/qgsd-bin/update-scoreboard.cjs` for scoreboard calls. Use `~/.claude/qgsd/bin/gsd-tools.cjs` for gsd-tools calls. Both are correct for their respective binaries.

**Installer evidence (HIGH confidence — direct code inspection):**
```javascript
// bin/install.js line 1729-1740
// Copy bin/*.cjs scripts to qgsd-bin/ (used by commands with absolute paths)
const binDest = path.join(targetDir, 'qgsd-bin');
fs.mkdirSync(binDest, { recursive: true });
// copies entry → qgsd-bin/entry for each .cjs file
```

```javascript
// bin/install.js line 1622-1630
// Copy qgsd skill with path replacement
const skillSrc = path.join(src, 'qgsd-core');
const skillDest = path.join(targetDir, 'qgsd');
// copies qgsd-core/* → ~/.claude/qgsd/*
// This includes qgsd-core/bin/gsd-tools.cjs → ~/.claude/qgsd/bin/gsd-tools.cjs
```

**INT-01 conclusion:** The current `$HOME/.claude/qgsd-bin/update-scoreboard.cjs` path in all three files is the correct canonical path. The "inconsistency" the audit flagged is between the two different installer targets (`qgsd-bin/` vs `qgsd/bin/`), which serve different binaries. Both are valid. INT-01 fix is therefore about ensuring the three workflow files all use the same quoting style and `$HOME` vs `~` expansion convention — not about changing the directory.

**INT-01 REVISED UNDERSTANDING:** Looking at the actual grep results:
- plan-milestone-gaps.md line 125: `node "$HOME/.claude/qgsd-bin/update-scoreboard.cjs"`
- execute-phase.md line 440: `node "$HOME/.claude/qgsd-bin/update-scoreboard.cjs"`
- discuss-phase.md line 261: `node "$HOME/.claude/qgsd-bin/update-scoreboard.cjs"`

All three already use identical text. The audit description says "v0.13-02 workflows use `$HOME/.claude/qgsd-bin/update-scoreboard.cjs`; other binaries in same workflows use `$HOME/.claude/qgsd/bin/`". The inconsistency is cross-binary (gsd-tools uses qgsd/bin, update-scoreboard uses qgsd-bin) — not within the update-scoreboard calls themselves. The fix the audit recommends is: "align all scoreboard calls to `$HOME/.claude/qgsd/bin/update-scoreboard.cjs`". However, `update-scoreboard.cjs` does NOT exist at `~/.claude/qgsd/bin/` — it only exists at `~/.claude/qgsd-bin/`. Moving it would require installer changes. The correct INT-01 fix is therefore: verify (and document) that `$HOME/.claude/qgsd-bin/update-scoreboard.cjs` is the canonical and correct path, ensure path replacement in installer works correctly, and update the audit's fix recommendation in VERIFICATION.md to reflect the correct canonical path.

**Planner decision needed:** INT-01 may be a documentation fix (confirm current path is correct) rather than a code change. This simplifies the fix.

### Anti-Patterns to Avoid

- **Deleting the quorum gate**: The user-gate text in Step 5 is AFTER the quorum approval gate was already added. Do not remove the quorum gate — only remove the bare user dialog line that precedes it.
- **Modifying gsd-tools path**: `~/.claude/qgsd/bin/gsd-tools.cjs` is correct for gsd-tools; do not conflate it with update-scoreboard.cjs.
- **Changing the installer**: Out of scope. INT-01 fix is in the workflow markdown files only.

## Common Pitfalls

### Pitfall 1: Confusing INT-01 audit recommendation with reality

**What goes wrong:** The audit says "Fix: align all scoreboard calls to `$HOME/.claude/qgsd/bin/update-scoreboard.cjs`". If executed literally, this would produce a broken path (update-scoreboard.cjs does NOT exist at qgsd/bin).

**Why it happens:** The audit was written to recommend alignment to one canonical prefix. The recommendation assumed qgsd/bin would be the canonical location. But the installer puts update-scoreboard.cjs in qgsd-bin, not qgsd/bin.

**How to avoid:** Verify the path exists on disk BEFORE changing any text. `ls ~/.claude/qgsd-bin/update-scoreboard.cjs` returns the file. `ls ~/.claude/qgsd/bin/update-scoreboard.cjs` does NOT. Keep the current qgsd-bin path — it is correct.

**Warning signs:** If grep for the old path returns 0 matches on all three files after "fixing", the path was never wrong.

**Revised INT-01 scope:** Since all three files already use the identical `$HOME/.claude/qgsd-bin/update-scoreboard.cjs` string, INT-01 may reduce to: (a) confirming no other update-scoreboard references use a different path, and (b) documenting the correct canonical path in VERIFICATION.md. The planner should evaluate whether a substantive text change is needed at all for INT-01.

### Pitfall 2: Removing quorum gate instead of user-dialog text (INT-02)

**What goes wrong:** Editor removes the wrong block — deletes the `**Quorum approval gate (QUORUM-01):**` section instead of the bare `Create these {X} phases?` line above it.

**Why it happens:** Both are in Step 5, close together in the file. Line 107 (user dialog) is inside a markdown code block (lines 77-108). The quorum gate starts at line 110.

**How to avoid:** The user-dialog line is INSIDE a fenced code block (` ``` `). The fix for line 107 is to remove that line from within the code block template — not delete the code block itself or the quorum gate section.

**Warning signs:** If `grep -n "Quorum approval gate"` returns no match after the edit, something was over-deleted.

### Pitfall 3: Stale success_criteria text changes meaning

**What goes wrong:** Leaving `User confirmed phase plan` in `<success_criteria>` causes a verifier agent to check for user confirmation that no longer happens (quorum replaced it). The verifier would report FAIL.

**How to avoid:** Replace with: `Quorum approved gap closure phases (GSD_DECISION marker present)` — matches the actual flow now in Step 5.
