# Phase v0.13-04 Plan 02: Fix INT-02 — Remove User-Gate Text from plan-milestone-gaps Summary

Removed three residual user-gate text fragments from `qgsd-core/workflows/plan-milestone-gaps.md` that would cause autonomous subagents to pause awaiting user input; quorum approval gate (QUORUM-01) left intact.

## What Was Built

`plan-milestone-gaps.md` had three stale text fragments from before QUORUM-01 was implemented. These fragments instructed subagents to ask the user for decisions that the quorum gate now handles automatically. The changes make the workflow safe for fully autonomous execution.

### Change 1 — Step 2 Priority Table (Task 1)

**Before:**
```
| `nice` | Ask user: include or defer? |
```

**After:**
```
| `nice` | Auto-defer in `--auto` mode; include otherwise |
```

The old text was a literal instruction to prompt the user. The new text describes the actual auto-mode behavior: nice gaps are deferred silently, interactive mode presents them.

### Change 2 — Step 5 Fenced Code Block Nice-to-Have Section (Task 2, Fix A)

**Before:**
```
{If nice-to-have gaps exist:}

### Deferred (nice-to-have)

These gaps are optional. Include them?
- {gap description}
- {gap description}

---

Create these {X} phases? (yes / adjust / defer all optional)
```

**After:**
```
{If nice-to-have gaps exist AND not running in --auto mode:}

### Deferred (nice-to-have)

These gaps are optional. Include them?
- {gap description}
- {gap description}

{If nice-to-have gaps exist AND running in --auto mode:}
Nice-to-have gaps deferred automatically (auto mode). Include manually if desired.

---
```

The conditional now makes auto-mode behavior explicit. The bare `Create these {X} phases?` user-dialog line was removed (it was replaced by the QUORUM-01 gate already in Step 5).

### Change 3 — success_criteria (Task 2, Fix B)

**Before:**
```
- [ ] User confirmed phase plan
```

**After:**
```
- [ ] Quorum approved gap closure phases (GSD_DECISION marker present in Step 5 output)
```

The old criterion referenced human confirmation; the new one references the GSD_DECISION marker that the quorum gate writes on APPROVED, making the criterion machine-verifiable.
