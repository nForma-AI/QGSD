Implement these exported functions:

1. **getLayerGitDiff(layer, root, commitCount)** — Extracts recent git diff for files related to a layer:
   - Define LAYER_FILE_MAP: mapping of layer keys to relevant file paths/globs (from RESEARCH.md)
   - Run `git diff HEAD~{commitCount} -- {paths}` (default commitCount=5)
   - Return truncated diff (max 2000 chars to fit in Haiku context window)
   - Fail-open: return empty string if git command fails

2. **buildClassificationPrompt(context)** — Pure function:
   - Takes `{ layer, deltas, gitDiffSummary, oscillationCount, currentResidual, previousResidual }`
   - Returns a structured prompt string for Haiku:
     ```
     You are an oscillation classifier for the nForma consistency solver.

     A layer's oscillation breaker has fired. Classify the root cause.

     Layer: {layer}
     Recent residual deltas: {deltas as comma-separated list}
     Current residual: {currentResidual}
     Previous residual: {previousResidual}
     Oscillation count: {oscillationCount}

     Recent git changes affecting this layer:
     {gitDiffSummary}

     Classify as exactly one of:
     - GENUINE_REGRESSION: The residual increase reflects real quality degradation (new bugs, removed tests, broken formal models)
     - MEASUREMENT_NOISE: The increase is an artifact of measurement (new requirements added, scope growth, tooling changes)
     - INSUFFICIENT_EVIDENCE: Cannot determine cause from available data (too few sessions, no relevant git changes)

     Respond with a JSON object:
     {"classification": "GENUINE_REGRESSION|MEASUREMENT_NOISE|INSUFFICIENT_EVIDENCE", "confidence": 0.0-1.0, "reasoning": "brief explanation"}
     ```

3. **parseClassificationResponse(responseText)** — Pure function:
   - Extracts JSON from Haiku response (may be wrapped in markdown code blocks)
   - Validates classification is one of the 3 valid values
   - Returns `{ classification, confidence, reasoning }` or defaults to `{ classification: "INSUFFICIENT_EVIDENCE", confidence: 0, reasoning: "Failed to parse classifier response" }`

4. **callHaiku(prompt)** — HTTP call to Anthropic API:
   - Read API key from ANTHROPIC_API_KEY environment variable
   - POST to https://api.anthropic.com/v1/messages
   - Model: claude-haiku-4-5-20251001 (same as nf-circuit-breaker.js)
   - Max tokens: 256, temperature: 0
   - 10-second timeout (matching circuit breaker pattern)
   - Returns response text or null on failure
   - Fail-open: if API key missing, network error, or timeout, return null
   - Log to stderr on failure: `[escalation-classifier] WARNING: Haiku call failed: {reason}`

5. **classifyEscalation(options)** — Main orchestration function:
   - Takes `{ layer, root, trendPath, verdictsPath }`
   - Reads verdicts to confirm layer is blocked
   - Reads trend window to extract delta series for the layer
   - Calls getLayerGitDiff() for context
   - Calls buildClassificationPrompt() then callHaiku()
   - Parses response via parseClassificationResponse()
   - Returns `{ layer, classification, confidence, reasoning, context: { deltas, gitDiffAvailable, oscillationCount } }`
   - Fail-open: if Haiku is unavailable, returns INSUFFICIENT_EVIDENCE with reasoning "Haiku classifier unavailable"

6. **detectNewlyBlocked(prevVerdicts, currVerdicts)** — Pure function:
   - Compares two verdict objects
   - Returns array of layer names that transitioned from `blocked: false` to `blocked: true`
   - Used by nf-solve.cjs to detect when to trigger classification

7. **CLI mode**: When run as `node bin/escalation-classifier.cjs --layer={layer} --project-root={root}`:
   - Runs classifyEscalation for the specified layer
   - Outputs JSON result to stdout

Module pattern: `module.exports = { classifyEscalation, buildClassificationPrompt, parseClassificationResponse, detectNewlyBlocked, getLayerGitDiff };`

IMPORTANT: The Haiku call is the ONLY external dependency. Everything else is pure computation or file I/O. The module must work completely when ANTHROPIC_API_KEY is not set (fail-open to INSUFFICIENT_EVIDENCE).
