Create bin/convergence-report.cjs following the established module pattern (see oscillation-detector.cjs, predictive-power.cjs).

Header: `'use strict';` with JSDoc comment referencing Requirements: INTG-02.

Import dependencies:
```javascript
const fs = require('fs');

const { readTrendWindow, LAYER_KEYS } = require('./oscillation-detector.cjs');
```

Implement these exported functions:

1. **generateSparkline(values)** — Pure function, no I/O:
   - Takes array of numeric values (residual series for a layer)
   - Unicode sparkline characters: `▁▂▃▄▅▆▇█` (8 levels, indices 0-7)
   - For -1 values (skipped/missing), use `·` (middle dot U+00B7)
   - If all non-negative values are equal, map to `▄` (middle level)
   - Otherwise: min/max normalize, index = Math.round((v - min) / (max - min) * 7)
   - Returns string of sparkline characters
   - Empty array returns empty string

2. **rankActionItems(verdicts, latestResiduals)** — Pure function:
   - Takes verdicts object (from oscillation-verdicts.json `layers` field) and latest residual per layer
   - Returns array of action items sorted by priority (descending)
   - Priority ranking rules:
     a. Blocked layers (oscillation breaker fired) — priority 100 + residual
     b. INCREASING trend layers — priority 80 + residual
     c. OSCILLATING trend layers — priority 60 + residual
     d. STABLE trend with residual > 0 (stalled) — priority 40 + residual
     e. DECREASING trend (making progress) — priority 0 (not actionable)
     f. INSUFFICIENT_DATA — priority 20 (needs more data points)
   - Each action item: `{ layer, residual, trend, priority, blocked, grace_period, action }` where action is a human-readable recommendation:
     - Blocked: "BLOCKED — oscillation detected. Run /nf:resolve {layer} for manual triage"
     - INCREASING: "Residual increasing — investigate recent changes to {layer} pipeline"
     - OSCILLATING: "Oscillating — cascade grace may resolve; monitor next 2 sessions"
     - STABLE (residual > 0): "Stalled at {N} — check if layer needs manual intervention"
     - INSUFFICIENT_DATA: "Insufficient data — need {5 - entry_count} more solve sessions"
   - Return only top N items (caller passes N, default 3)

3. **formatConvergenceSection(options)** — Orchestration function:
   - Takes `{ root, maxItems }` (maxItems defaults to 3)
   - Reads solve-trend.jsonl via readTrendWindow(trendPath, 20)
   - Reads oscillation-verdicts.json from `.planning/formal/oscillation-verdicts.json`
   - Reads latest residual from the last trend entry
   - For each layer in LAYER_KEYS: extract residual series, call generateSparkline()
   - Calls rankActionItems() with verdicts and latest residuals
   - Returns formatted string with three sections:

   Section 1 — Trend Sparklines:
   ```
   ─ Convergence Trends (last 20 sessions) ────────────────
   Layer               Trend          Sparkline
   ─────────────────────────────────────────────────────────
   r_to_f              DECREASING     ████▇▅▃▂▁▁▁
   f_to_t              STABLE         ▄▄▄▄▄▄▄▄▄▄
   ...
   ```
   Only show layers that have data (skip layers where all entries are -1).
   Use the trend from verdicts if available, otherwise "NO_DATA".

   Section 2 — Oscillation Status:
   ```
   ─ Oscillation Status ───────────────────────────────────
   Layer               Credits  Blocked  Grace
   ─────────────────────────────────────────────────────────
   r_to_f              1/1      no       no
   f_to_t              0/1      YES      no
   ```
   Only show layers with oscillation_count > 0 OR blocked OR grace_period.
   If no layers have issues, show: "All layers stable — no oscillation detected"

   Section 3 — Action Items:
   ```
   ─ Top 3 Action Items ───────────────────────────────────
   1. [f_to_t] BLOCKED — oscillation detected. Run /nf:resolve f_to_t for manual triage
   2. [c_to_f] Residual increasing — investigate recent changes to c_to_f pipeline
   3. [t_to_c] Stalled at 5 — check if layer needs manual intervention
   ```

   If no action items (all layers converged or no data), show:
   "No action items — all tracked layers are converging or converged"

   Fail-open: if any file read fails, return a minimal section with a note:
   "Convergence data unavailable — run at least 5 solve sessions to enable trend tracking"

4. **CLI mode**: When run as `node bin/convergence-report.cjs --json`, output the raw data as JSON (for machine consumption). When run without --json, output the formatted section to stdout. Use standard CLI pattern:
   ```javascript
   if (require.main === module) {
     const args = process.argv.slice(2);
     // Parse --project-root, --json, --max-items flags
     // Call formatConvergenceSection() and output
   }
   ```
