# Phase v0.12-04 Plan 02: TLA+ Spec Authoring (GAP-1 + GAP-5) Summary

Two TLA+ formal specifications authored for the circuit breaker algorithm verification phase: QGSDOscillation.tla (GAP-1: run-collapse oscillation detection correctness) and QGSDConvergence.tla (GAP-5: resolvedAt write-once, log-before-delete ordering, Haiku failure safety).

## What Was Built

### Task 1: QGSDOscillation.tla + MCoscillation.cfg (GAP-1)

`formal/tla/QGSDOscillation.tla` models the two-stage run-collapse oscillation detection algorithm from `hooks/qgsd-circuit-breaker.js`. The spec abstracts file sets as labels from a finite alphabet `{A, B, C}`, allowing TLC to verify the counting correctness property without modeling actual file names.

**State variables:**
- `commits` — Seq of labels (abstract file-set identifiers)
- `runs` — Run-collapse of commits (consecutive duplicates merged)
- `flagCount` — [label -> Nat] count of run-groups per label
- `netChange` — Integer diff sum (nondeterministically chosen; models `hasReversionInHashes` result)
- `flagged` — BOOLEAN: oscillation detected
- `algorithmDone` — BOOLEAN: algorithm has completed all steps

**Key invariant (GAP-1):** `OscillationFlaggedCorrectly` asserts `flagged <=> (\E l \in Labels : flagCount[l] >= Depth /\ netChange <= 0)`.

**Liveness:** `AlgorithmTerminates == <>(algorithmDone = TRUE)` with `WF_vars(EvaluateFlag) /\ WF_vars(CollapseRuns)` in Spec.

`formal/tla/MCoscillation.cfg` configures TLC with `Labels = {A, B, C}`, `Depth = 3`, `CommitWindow = 5`, INVARIANT TypeOK + OscillationFlaggedCorrectly, PROPERTY AlgorithmTerminates, CHECK_DEADLOCK FALSE, and no SYMMETRY directive.

### Task 2: QGSDConvergence.tla + MCconvergence.cfg (GAP-5)

`formal/tla/QGSDConvergence.tla` models the PostToolUse Haiku convergence check state transitions, proving three persistence invariants about the circuit breaker state file and oscillation log.

**State variables:**
- `logWritten` — BOOLEAN: oscillation log entry has `resolvedAt` set
- `stateDeleted` — BOOLEAN: `.claude/circuit-breaker-state.json` removed
- `haikuVerdict` — One of `"YES"`, `"NO"`, `"UNAVAILABLE"`

**Key invariants (GAP-5):**
- `ResolvedAtWriteOnce`: `[][logWritten = TRUE => logWritten' = TRUE]_vars` — write-once temporal invariant
- `LogBeforeDelete`: `stateDeleted => logWritten` — ordering invariant
- `HaikuUnavailableNoCorruption`: Haiku unavailability causes no state mutation

**Liveness:** `ConvergenceEventuallyResolves == <>(logWritten = TRUE)` with `WF_vars(HaikuReturnsYES)` in Spec.

`formal/tla/MCconvergence.cfg` declares all 4 INVARIANT entries, PROPERTY ConvergenceEventuallyResolves, CHECK_DEADLOCK FALSE.
