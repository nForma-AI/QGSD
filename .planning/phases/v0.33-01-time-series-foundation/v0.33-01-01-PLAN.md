<task type="auto">
  <name>Task 1: Implement JSONL trend append with scope-growth detection in nf-solve.cjs</name>
  <files>bin/nf-solve.cjs</files>
  <action>
Add two new functions to bin/nf-solve.cjs, placed immediately above the `main()` function or in the utility section near `persistSessionSummary()`:

1. **`readGateSummary()`** -- Reads gate scores from the 3 gate JSON files in `.planning/formal/gates/`. Handle each gate field name explicitly:
   - gate-a-grounding.json uses `grounding_score`
   - gate-b-abstraction.json uses `gate_b_score`
   - gate-c-validation.json uses `gate_c_score`
   Each gate entry returns `{ score: Number, target_met: Boolean }`. Wrap each file read in try/catch (fail-open, return null for that gate on error).

2. **`appendTrendEntry(finalResidual, converged, iterations)`** -- Appends a single JSON line to `.planning/formal/solve-trend.jsonl`. Implementation:
   - Read requirement count from `.planning/formal/requirements.json` (the `requirements` array length, currently 335). Fail-open to 0 if file missing.
   - Call `readGateSummary()` for gate data.
   - Build `per_layer` object by iterating the 19 known layer keys: `r_to_f, f_to_t, c_to_f, t_to_c, f_to_c, r_to_d, d_to_c, p_to_f, c_to_r, t_to_r, d_to_r, l1_to_l2, l2_to_l3, l3_to_tc, per_model_gates, git_heatmap, git_history, formal_lint, hazard_model`. For each key, extract `finalResidual[key].residual` if it exists, otherwise -1 (preserves fast-mode skipped layers per research Pitfall 2).
   - Determine `scope_change` by reading the last valid line of solve-trend.jsonl (skip malformed lines by parsing in reverse). Compare current `requirement_count` vs previous entry's `requirement_count`:
     - First entry or no previous: `null`
     - Increased: `"SCOPE_GROWTH"`
     - Same: `"STABLE"`
     - Decreased: `"SCOPE_REDUCTION"`
   - Include `fast_mode` boolean field (check if global `FAST_FLAG` or equivalent is set) per research Open Question 3.
   - Build entry object matching the schema in RESEARCH.md Pattern 1.
   - Append with `fs.appendFileSync(trendPath, JSON.stringify(entry) + '\n')`.
   - Wrap entire function in try/catch, log warning to stderr on failure (fail-open pattern).

3. **Call `appendTrendEntry(finalResidual, converged, iterations)`** in `main()` after the solve-state.json write block (~line 3494, after the try/catch that writes solve-state.json) and before `persistSessionSummary()`. This ensures trend data is written even if session persistence fails.

Important: Do NOT modify the solve-state.json format (per STATE.md decision). The JSONL file is a separate artifact.
