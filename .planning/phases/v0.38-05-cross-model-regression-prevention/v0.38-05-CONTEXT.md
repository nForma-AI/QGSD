# Phase v0.38-05: Cross-Model Regression Prevention - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Fixes verified against one formal model are also checked against proximity-neighbor models (from proximity-index.json) to prevent cross-model regressions. Extends model-driven-fix.md Phase 5, adds scoped verification to run-formal-verify.cjs, and enriches bug-model-gaps.json with post-fix verification results.

</domain>

<decisions>
## Implementation Decisions

### Neighbor hop distance and cap
- 2-hop neighbors from proximity-index.json (per success criteria — fixed)
- Cap at max 10 neighbor models to prevent runaway verification on highly connected models
- Cap is config-backed (consistent with existing per-layer configurable limits)
- If a model has more than 10 neighbors at 2 hops, take the 10 with highest proximity score

### Regression handling behavior
- Warn and report regressions but do NOT block the fix by default (fail-open pattern)
- Regressions array in post_fix_verification captures which neighbor models failed
- User can re-run with `--strict` flag to gate fix completion on all neighbor models passing
- Default behavior: fix proceeds, regressions logged for manual review

### Verification results format
- Summary table in fix output: model_pass (boolean), neighbor_models_pass (boolean), regressions count
- JSON detail in bug-model-gaps.json entries: full post_fix_verification object with per-model results
- Matches existing FORMAL verdict row pattern from Phase v0.38-02

### Scope flag behavior
- `--scope=model1,model2` runs only specified models (explicit list)
- No auto-expansion shortcut (keep interface simple — model-driven-fix Phase 5 handles neighbor expansion internally)
- Default when no `--scope` given: run the full suite (existing behavior preserved)
- model-driven-fix.md Phase 5 automatically expands scope to include 2-hop neighbors before calling run-formal-verify.cjs

### Claude's Discretion
- Proximity score threshold for neighbor selection when capping at 10
- Internal data structure for post_fix_verification results
- Error handling when proximity-index.json is missing or stale
- Timeout per neighbor model checker (recommend reusing existing 30s per-model timeout)

</decisions>
