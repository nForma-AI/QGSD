## Decisions Made

1. **Inline JSON Schema validation:** Implemented custom validateEnvelope() function instead of using ajv library. Rationale: project has zero schema validation dependencies; task-envelope.cjs already demonstrates this pattern; keeps deployment lightweight.

2. **Deterministic timestamps:** Added --deterministic flag using fixed timestamp '2026-03-01T20:32:24.000Z' for testing. Content hash computed from requirements array only (not timestamp), enabling verification that re-runs with different timestamps produce identical hashes.

3. **Frozen envelope protection:** Check for existing frozen_at before overwriting output file. Prevents accidental data loss and enforces amendment workflow (ENV-04) for modifications.

4. **Sorted JSON output:** Keys in alphabetical order (aggregated_at, content_hash, frozen_at, requirements, schema_version, source) ensures byte-identical output on re-runs, enabling reproducible verification.
