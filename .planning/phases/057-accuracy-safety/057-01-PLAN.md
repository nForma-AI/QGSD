<objective>
Add Layer 2.5 to formal-scope-scan.cjs: walk getCallers() backward from --files to discover formal models affected by transitive call-graph relationships that file-name matching alone would miss.

Purpose: A utility function change may not share a name with any formal model, but if that utility is called by a file in a formal model's source_files, the model needs re-verification. The existing proximity index (Layer 2) discovers graph-connected modules, but only when those modules are in proximity-index.json. Layer 2.5 uses live call-graph data from coderlm to close this gap.

Output: Modified bin/formal-scope-scan.cjs with Layer 2.5 inserted between Layer 2 and Layer 3. New tests for Layer 2.5 in bin/formal-scope-scan.test.cjs.

**Layer 2.5 algorithm:**

```javascript
function discoverViaCallGraph(files, matchedModules, adapter) {
  // Returns array of new match objects (not in matchedModules already)
  // adapter: createAdapter() instance or null
  const newMatches = [];
  if (!adapter || !files || files.length === 0) return newMatches;

  let healthy = false;
  try {
    const h = adapter.healthSync();
    healthy = h.healthy;
  } catch (_) {}
  if (!healthy) return newMatches;

  for (const changedFile of files) {
    let callers;
    try {
      const result = adapter.getCallersSync('', changedFile);
      if (result.error || !Array.isArray(result.callers)) continue;
      callers = result.callers; // array of file paths that call changedFile
    } catch (_) {
      continue; // fail-open: skip this file on any error
    }

    // For each caller file, check if it matches any formal module's source_files globs
    for (const callerFile of callers) {
      const normalizedCaller = callerFile.replace(/\\/g, '/');
      for (const mod of getAllModules()) {
        if (matchedModules.has(mod)) continue;
        const scopePath = path.join(SPEC_DIR, mod, 'scope.json');
        let scope;
        try { scope = JSON.parse(fs.readFileSync(scopePath, 'utf8')); } catch (_) { continue; }
        if (scope.source_files && matchesSourceFiles([normalizedCaller], scope.source_files)) {
          matchedModules.add(mod);
          newMatches.push({
            module: mod,
            path: '.planning/formal/spec/' + mod + '/invariants.md',
            matched_by: 'call_graph',
            discovered_via: changedFile + ' -> ' + normalizedCaller
          });
        }
      }
    }
  }
  return newMatches;
}
```
