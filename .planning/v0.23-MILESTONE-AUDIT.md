## Integration Issues

### ISSUE-1 (Medium): Keyword-Match Algorithm Mismatch

- **Where:** plan-phase.md Step 4.5 vs execute-phase.md verify_phase_goal
- **Problem:** Different field extraction (`.phase_name` vs `.goal`), different separators (`' -/'` vs `' -'`), different match direction
- **Risk:** Phase with goal differing from name or containing `/` → different FORMAL_SPEC_CONTEXT at planning vs execution time
- **Fix:** Standardize to `.goal // .phase_name` + `' -/'` separator + bidirectional match in both phases

### ISSUE-2 (Medium): Missing .goal Field Asymmetry

- **Where:** execute-phase.md `.goal // empty` vs plan-phase.md `.phase_name`
- **Problem:** Phases without `.goal` silently skip formal checking in execute-phase
- **Fix:** Standardize fallback to `.goal // .phase_name` in both places
