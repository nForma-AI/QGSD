## Unsatisfied Requirements

### LIVE-02 — Liveness fairness check partial

`detectLivenessProperties()` implemented in `run-tlc.cjs` but only called there. Four other TLC runners execute liveness configs without the fairness check:

| Runner | Config | Unchecked PROPERTY |
|--------|--------|--------------------|
| run-oscillation-tlc.cjs | MCoscillation.cfg | AlgorithmTerminates |
| run-oscillation-tlc.cjs | MCconvergence.cfg | ConvergenceEventuallyResolves |
| run-breaker-tlc.cjs | MCbreaker.cfg | MonitoringReachable |
| run-protocol-tlc.cjs | MCdeliberation.cfg | ProtocolTerminates |
| run-protocol-tlc.cjs | MCprefilter.cfg | PreFilterTerminates |
| run-account-manager-tlc.cjs | MCaccount-manager.cfg | IdleReachable |

All 6 have companion `invariants.md` (LIVE-01 satisfied). **Fix:** add `detectLivenessProperties()` call to each of the 4 runners before their TLC invocation.

### MCPENV-02 — TLA+ MCP spec not in automated pipeline

`QGSDMCPEnv.tla` + `MCMCPEnv.cfg` exist and are well-formed. Not automated anywhere:
- Not in `run-formal-verify.cjs` STEPS
- No CI workflow step
- Not in `run-tlc.cjs` SURFACE_MAP
- `formal/spec/mcp-calls/invariants.md` absent

**Fix:** (1) Create `formal/spec/mcp-calls/invariants.md` with EventualDecision fairness declaration. (2) Add MCMCPEnv.cfg to SURFACE_MAP. (3) Add to run-formal-verify.cjs STEPS + CI workflow.

### MCPENV-04 — PRISM MCP availability model three bugs

1. **Dead export:** `module.exports` block placed after unconditional `process.exit()` — unreachable via `require()`. Fix: use `require.main === module` guard (same pattern as run-tlc.cjs).
2. **Composite-key corruption:** `slot === 'claude'` exclusion misses composite keys like `claude-1:deepseek-ai/DeepSeek-V3.2`; `.replace(/-/g, '_')` produces invalid PRISM constant names. Fix: exclude keys containing `:`.
3. **Not in pipeline:** No CI step, no run-formal-verify.cjs entry. Fix: add after standard PRISM step.
