## Success Criteria Verification

### 1. TLA+ spec auto-generated from plan truths
**Status: VERIFIED**
- `bin/generate-proposed-changes.cjs` reads `must_haves: truths:` from PLAN.md YAML frontmatter
- Generates `ProposedChanges.tla` in `{phaseDir}/.formal/` (phase-scoped, not `.formal/tla/scratch/`)

### 2. TLC runs with iterative feedback
**Status: VERIFIED**
- `bin/run-phase-tlc.cjs` invokes TLC via `spawnSync` with 60s timeout
- Parses violation patterns (Invariant/Property violations, Error lines)
- `iterativeVerify` performs single-attempt flow: generate spec -> run TLC -> return structured result
- `formatTlcFeedback` maps violated ReqNN back to original truth strings via zero-padded index parsing
- Fail-open: missing Java or tla2tools.jar returns structured error, never crashes
- Integration test confirmed: stubs (all TRUE) pass TLC verification

### 3. Quorum slots receive formal evidence
**Status: VERIFIED**
- `bin/quorum-formal-context.cjs` generates plain-English formal spec summaries (NOT raw TLA+ code)
- Special character sanitization: angle brackets -> square brackets, newlines -> spaces, backtick wrapping
- `generateVerificationResult` maps PASS/FAIL/INCONCLUSIVE states
- `buildFormalEvidenceBlock` produces `=== Formal Evidence ===` block for prompt injection
- `getFormalContext` convenience API combines all steps
