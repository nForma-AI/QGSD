**TLA+ module — formal/tla/QGSDStopHook.tla:**

Use the template from RESEARCH.md verbatim with these exact variables, actions, and properties:
- Variables: hasCommand, hasQuorumEvidence, decision ("BLOCK"/"PASS"/"UNDECIDED"), algorithmDone
- Init: all FALSE/UNDECIDED/FALSE
- Actions: DetectCommand (nondeterministic hasCommand'), DetectQuorumEvidence (nondeterministic hasQuorumEvidence'), MakeDecision (sets decision from IF/THEN/ELSE logic + algorithmDone=TRUE)
- Safety: SafetyInvariant1 (BLOCK => hasCommand), SafetyInvariant2 (BLOCK => ~hasQuorumEvidence), SafetyInvariant3 (PASS => hasCommand /\ hasQuorumEvidence)
- Liveness: LivenessProperty1 (<>algorithmDone), LivenessProperty2 (hasQuorumEvidence => <>(decision="PASS")), LivenessProperty3 (hasCommand /\ ~hasQuorumEvidence => <>(decision="BLOCK"))
- Spec: Init /\ [][Next]_vars /\ WF_vars(MakeDecision) /\ WF_vars(DetectCommand) /\ WF_vars(DetectQuorumEvidence)

The CONSTANTS section has a single constant MaxTurnLines (not used in logic — retained for extensibility, set to model value 500).

**TLC config — formal/tla/MCStopHook.cfg:**

Model on MCoscillation.cfg as a template. Use:
```
\* formal/tla/MCStopHook.cfg
\* TLC model for QGSDStopHook.tla — Stop hook decision logic.
\* Checks safety invariants + liveness properties for planning command gate.
\*
\* Run: node bin/run-stop-hook-tlc.cjs MCStopHook  (requires Java >=17 + tla2tools.jar)
\* Note: -workers 1 required for liveness check.
\*
\* IMPORTANT: Do NOT add a FAIRNESS line here. Fairness is declared via WF_vars()
\* inside the Spec formula in QGSDStopHook.tla (e.g., WF_vars(MakeDecision)).
\* Adding a separate FAIRNESS line in the .cfg while WF_vars is in Spec causes
\* TLC liveness verification errors from conflicting fairness declarations.
\* This matches the MCoscillation.cfg pattern — see that file for reference.
SPECIFICATION Spec
CONSTANTS
    MaxTurnLines = 500
INVARIANT TypeOK
INVARIANT SafetyInvariant1
INVARIANT SafetyInvariant2
INVARIANT SafetyInvariant3
PROPERTY LivenessProperty1
PROPERTY LivenessProperty2
PROPERTY LivenessProperty3
CHECK_DEADLOCK FALSE
```

FAIRNESS is declared in the Spec formula itself (WF_vars(...)) in the .tla file — do NOT add a separate FAIRNESS line in the .cfg. Verify this matches MCoscillation.cfg (which has no FAIRNESS line — fairness is entirely in the Spec formula inside QGSDOscillation.tla).
