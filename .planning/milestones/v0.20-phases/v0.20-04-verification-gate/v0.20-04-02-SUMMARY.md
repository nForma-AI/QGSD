## Implementation Notes

### bin/verify-formal-results.cjs

- `parseNDJSON(ndjsonPath)` — reads NDJSON file with `fs.readFileSync`, splits by `\n`, skips empty lines, uses `try/catch` per line to skip malformed JSON (fail-open). Returns empty array if file missing/unreadable.
- `groupByFormalism(results)` — uses `result.formalism` as dynamic key (no hardcoded list). Initializes `{pass, fail, warn, inconclusive}` counters to 0 for each formalism. Only increments known result types.
- `generateFVSection(grouped, command, timestamp)` — determines overall status (fail > inconclusive > pass). Renders per-formalism tables and a summary table with totals. Returns markdown string starting with `## Formal Verification`.
