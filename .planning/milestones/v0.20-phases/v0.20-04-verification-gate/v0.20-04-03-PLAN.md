**Step: Formal Verification Gate (VERIFY-01)**

After goal-achievement verification completes and before writing VERIFICATION.md, run the TLA+ formal verification step. This step is fail-open â€” FV unavailability must not prevent VERIFICATION.md from being written.

```bash
# Run TLA+ formal verification â€” fail-open (non-zero exit OK)
FV_EXIT_CODE=0
node bin/run-formal-verify.cjs --only=tla 2>/dev/null || FV_EXIT_CODE=$?

# Generate ## Formal Verification section from check-results.ndjson
FV_SECTION=""
FV_TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

node -e "
  const { parseNDJSON, groupByFormalism, generateFVSection } = require('./bin/verify-formal-results.cjs');
  const ndjsonPath = '.formal/check-results.ndjson';
  const results = parseNDJSON(ndjsonPath);
  const grouped = groupByFormalism(results);
  if (Object.keys(grouped).length > 0) {
    const section = generateFVSection(grouped, 'run-formal-verify --only=tla', process.env.FV_TIMESTAMP || new Date().toISOString());
    process.stdout.write(section);
  } else {
    process.stdout.write('## Formal Verification\n\nNo formal verification results found. Run \`node bin/run-formal-verify.cjs\` to generate results.\n');
  }
" 2>/dev/null > /tmp/fv_section_$$.md || true

if [ -s /tmp/fv_section_$$.md ]; then
  FV_SECTION=$(cat /tmp/fv_section_$$.md)
  echo "[FV] Formal verification section generated ($(wc -l < /tmp/fv_section_$$.md) lines)"
else
  FV_SECTION="## Formal Verification\n\nFormal verification did not produce results (tool unavailable or no checks configured)."
  echo "[FV] Formal verification unavailable â€” recording absence in VERIFICATION.md (fail-open)"
fi
rm -f /tmp/fv_section_$$.md
```
