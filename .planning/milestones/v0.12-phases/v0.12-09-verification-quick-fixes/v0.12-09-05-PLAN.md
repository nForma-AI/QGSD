# v0.12-09-05 — PET-03: Fix permanently-inert deadlock condition

## Problem

`bin/generate-petri-net.cjs` lines 21-34:

```javascript
const QUORUM_SLOTS    = ['gemini', 'opencode', 'copilot', 'codex', 'claude'];
const MIN_QUORUM_SIZE = Math.ceil(QUORUM_SLOTS.length / 2);  // = 3

// PET-03: structural deadlock check (pure logic — before any rendering)
if (MIN_QUORUM_SIZE > QUORUM_SLOTS.length) {
  process.stderr.write(
    '[generate-petri-net] WARNING: Structural deadlock detected.\n' + ...
  );
}
```

`MIN_QUORUM_SIZE = Math.ceil(5/2) = 3`. `QUORUM_SLOTS.length = 5`.

`3 > 5` is **always false** with hardcoded constants. The deadlock warning can never fire.
The check is permanently inert — it will never protect against a misconfiguration where
the quorum threshold exceeds the number of available slots.

**Source:** v0.12 audit, PET-03.

## Fix

Add an optional `--min-quorum=N` CLI argument. When provided, override `MIN_QUORUM_SIZE`
with `N`. This makes the deadlock check actually exercisable at runtime:

```bash
node bin/generate-petri-net.cjs --min-quorum=6
# MIN_QUORUM_SIZE = 6, QUORUM_SLOTS.length = 5
# 6 > 5 = true → deadlock WARNING fires to stderr
# DOT still written (warning does not exit 1)
```

Default behavior (no flag) remains unchanged:
- `MIN_QUORUM_SIZE = Math.ceil(QUORUM_SLOTS.length / 2) = 3`
- `3 > 5 = false` → no warning (identical to current behavior)
- Zero regression

## Implementation

### 1. Add CLI argument parsing (before the deadlock check)

Insert after line 20 (after `MIN_QUORUM_SIZE` declaration):

```javascript
// Optional --min-quorum=N override (makes PET-03 deadlock check exercisable at runtime)
const minQuorumArg = process.argv.slice(2).find(a => a.startsWith('--min-quorum='));
const effectiveMinQuorum = minQuorumArg
  ? parseInt(minQuorumArg.split('=')[1], 10)
  : MIN_QUORUM_SIZE;
```

```

```
