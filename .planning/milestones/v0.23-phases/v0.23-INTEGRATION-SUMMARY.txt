ISSUE 1: KEYWORD MATCHING ALGORITHM MISMATCH
────────────────────────────────────────────
Severity:    MEDIUM
Location:    plan-phase.md (line 80) vs execute-phase.md (line 374)
Status:      BLOCKS v0.23-03 ROADMAPPER INTEGRATION
Risk:        Inconsistent formal scope detection between planning and execution

Details:
  • plan-phase uses `.phase_name` → execute-phase uses `.goal` (different fields)
  • plan-phase splits on ' -/' (3 separators) → execute-phase splits on ' -' (2)
  • plan-phase does bidirectional match → execute-phase does unidirectional match
  • Result: If phase goal differs from name, scope detection differs

Example: Phase "TUI/Navigation" with goal "Navigate TUI"
  - plan-phase would match: tui, navigation, gui (3 keywords from name)
  - execute-phase would match: navigate, tui (2 keywords from goal, no /)

Impact on WFI-01 (plan scope) and WFI-03 (execute scope):
  - Planner may include invariants not checked at execution
  - Executor may skip checks planner prepared for

Recommendation BEFORE v0.23-03:
  [PRIORITY: HIGH]
  1. Standardize to use `.goal` field in BOTH phases
  2. Standardize keyword split to ' -/' in BOTH
  3. Implement bidirectional match in BOTH (safer for substring matches)
  4. Test with phases containing / or - in goal field
  5. Update v0.23-01 and v0.23-02 to use consistent algorithm

Affected Requirements: WFI-01, WFI-03
