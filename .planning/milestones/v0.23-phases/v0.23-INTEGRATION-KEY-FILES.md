**Issue 1: Algorithm Mismatch**
```

  for KEYWORD in $(echo "$PHASE_DESC_LOWER" | tr ' -/' '\n' | grep -v '^$'); do
    if echo "$MODULE_LOWER" | grep -qF "$KEYWORD" || 
       echo "$KEYWORD" | grep -qF "$MODULE_LOWER"; then  ← bidirectional

execute-phase.md (line 374):
  PHASE_DESC=$(jq -r '.goal // empty')
  for KEYWORD in $(echo "$DESC_LOWER" | tr ' -' '\n' | grep -v '^$'); do  ← 2 separators only
    if echo "$MODULE_LOWER" | grep -qF "$KEYWORD"; then  ← unidirectional
```

**Impact:** If phase name contains `/` or goal differs from name, scope differs.
**Status:** BLOCKS v0.23-03 (roadmapper must use consistent algorithm)
```

### Forward Compatibility (WFI-04 + ENF-01 + ENF-02)

**FORMAL_CHECK_RESULT JSON contract:**
- `passed: N` — checks passed
- `failed: N` — checks failed (triggers counterexample_found)
- `skipped: N` — checks skipped (triggers tooling_absent if failed==0)
- `counterexamples: []` — list of failed module:tool pairs

**Parsing in qgsd-verifier Step 8.5:** ✓ COMPATIBLE
