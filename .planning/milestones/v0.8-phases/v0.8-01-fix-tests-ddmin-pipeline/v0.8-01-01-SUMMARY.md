# Phase v0.8-01 Plan 01: Add --run-cap Flag and Write fix-tests.md Phases 1-2 Summary

## What Was Built

### Task 1: `--run-cap N` flag on `maintain-tests ddmin`

Added `--run-cap N` CLI option (integer, default 50) to `maintain-tests ddmin`:
- Argument parsing: `args.indexOf('--run-cap')` → `parseInt(args[runCapIdx + 1], 10) : 50`
- `cmdMaintainTestsDdmin` now accepts `runCap` in options destructuring
- `const RUN_CAP = parseInt(runCap, 10) || 50;` — replaces hard-coded 50
- Updated doc comment in function and header usage string
- Backward compatible: callers without `--run-cap` still get 50
- Tested: `--run-cap 200` accepted without error; 136 passing tests, 3 pre-existing failures (unrelated)

### Task 2: fix-tests.md complete rewrite (4-phase ddmin pipeline)

Completely replaced the batch-based workflow with a 4-phase ddmin pipeline:

**Phase 1 — ddmin Isolation:**
- Step 1.1: Activity set
- Step 1.2: Discover tests (`maintain-tests discover`)
- Step 1.3: Full-suite pinned run (seed=42, `--size 9999` single mega-batch)
- Step 1.4: SHA-256 failure signature computation (inline Python) with 5-rule normalization
- Step 1.5: Flakiness pre-filter (N=10 reruns, STABLE if fail_count >= 8/10)
- Step 1.6: Per-test ddmin with `--run-cap 200` (cap at 100 stable tests)
- Step 1.7: Dependency graph construction + cycle detection (Tarjan SCC + Kahn's topological sort)
- Step 1.8: State save (schema_version 2)

**Phase 2 — Triage Report + Quorum Approval:**
- Step 2.1: Generate triage report (6 sections: independent failures, pollution chains, cycle groups, flaky, fix order, unanalyzed)
- Step 2.2: Hard quorum gate via `qgsd-quorum-orchestrator` (APPROVE/BLOCK)
- Step 2.3: State save with `pipeline_phase = "fixing"`

**Phase 3 — Sequential Fixing (also included):**
- Cycle group atomic handling before fix loop
- Per-fix: state diff capture, AI proposal, quorum approval, fix application, regression test, commit
- Post-fix full-suite signature diff with immediate revert + re-ddmin on regression

**Phase 4 — Final Verification (also included):**
- Final full-suite run, signature diff against original baseline
- Final report generation (fix log audit trail)
- Advisory quorum review (APPROVE/NOTE/BLOCK)
- Terminal summary banner
