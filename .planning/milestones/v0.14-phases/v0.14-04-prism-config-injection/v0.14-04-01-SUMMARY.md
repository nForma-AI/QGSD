# Plan v0.14-04-01 SUMMARY: PRISM Scoreboard Rate Injection

## What Was Built

Modified `bin/run-prism.cjs` to automatically read `.planning/quorum-scoreboard.json` at invocation time and inject empirical TP and unavailability rates as PRISM `-const` parameters.

## Changes Made

**bin/run-prism.cjs:**
- Added `readScoreboardRates(cwd)` helper that reads scoreboard, computes per-slot TP/UNAVAIL rates across SLOTS = ['gemini', 'opencode', 'copilot', 'codex'], and returns aggregate mean values
- Four fallback paths (all return conservative priors `0.85/0.15`): scoreboard absent, parse failure, empty rounds array, no slot data
- Added `callerOverridesTp` / `callerOverridesUnavail` guards (same pattern as `run-oauth-rotation-prism.cjs`) — caller-supplied `-const tp_rate=X` or `-const unavail=Y` take precedence
- Injects `-const tp_rate=X -const unavail=Y` before `extraArgs` in `prismArgs` (so caller args follow and win if duplicated)
- Logs `[run-prism] Injected from scoreboard: tp_rate=X unavail=Y` when empirical rates used (only shows rates that were actually injected, not overridden ones)
- Updated header comment to reference PRISM-01 and PRISM-02

## Verification

```
node --check bin/run-prism.cjs   → SYNTAX OK
node bin/run-prism.cjs 2>&1      → [run-prism] Injected from scoreboard: tp_rate=0.5056 unavail=0.4604
cd /tmp && node bin/run-prism.cjs 2>&1 → [run-prism] No scoreboard found — using conservative priors
node bin/run-prism.cjs -const tp_rate=0.99 2>&1 → [run-prism] Injected from scoreboard: unavail=0.4604 (tp_rate caller-overridden)
```
