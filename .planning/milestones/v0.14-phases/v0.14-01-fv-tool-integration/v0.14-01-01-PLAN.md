# Plan v0.14-01-01 — Commit Untracked FV Artifacts + Wire CI

## Goal

Bring all untracked formal verification tools and formal artifacts into source control, and update the CI workflow so it exercises the full `run-formal-verify.cjs` master runner end-to-end.

## Context

The following files exist on disk but are not tracked by git:
- `bin/xstate-to-tla.cjs` — XState → TLA+ transpiler
- `bin/run-formal-verify.cjs` — master runner (20 steps: generate + TLA+ + Alloy + PRISM + Petri)
- `bin/run-oauth-rotation-prism.cjs` — PRISM runner for oauth-rotation model
- `bin/run-account-pool-alloy.cjs` — Alloy runner for account-pool-structure spec
- `bin/run-account-manager-tlc.cjs` — TLC runner for QGSDAccountManager spec
- `bin/account-manager.cjs` — account manager implementation (source of truth for TLA+ spec)
- `.formal/alloy/account-pool-structure.als` — Alloy model
- `.formal/petri/account-manager-petri-net.dot` — Petri net DOT source
- `.formal/petri/account-manager-petri-net.svg` — Petri net SVG render
- `.formal/tla/MCaccount-manager.cfg` — TLC model config for account-manager
- `.formal/tla/QGSDAccountManager.tla` — TLA+ spec for account manager

.gitignore safety audit:
- `.formal/petri/quorum-petri-net.dot` and `quorum-petri-net.svg` are gitignored (quorum ones only)
- `account-manager-petri-net.dot` and `.svg` are NOT in .gitignore — safe to commit
- `.formal/alloy/org.alloytools.alloy.dist.jar`, `.formal/tla/tla2tools.jar`, `.formal/prism/rates.const` are gitignored — none of the above files match

CI gap: `.github/workflows/formal-verify.yml` is already committed but does NOT invoke `run-formal-verify.cjs` or `npm test`. It calls individual tool steps manually. This plan adds a master runner step.

## Tasks

### Task 1 — git add all untracked FV files

Stage for commit:
```
git add bin/xstate-to-tla.cjs \
        bin/run-formal-verify.cjs \
        bin/run-oauth-rotation-prism.cjs \
        bin/run-account-pool-alloy.cjs \
        bin/run-account-manager-tlc.cjs \
        bin/account-manager.cjs \
        .formal/alloy/account-pool-structure.als \
        .formal/petri/account-manager-petri-net.dot \
        .formal/petri/account-manager-petri-net.svg \
        .formal/tla/MCaccount-manager.cfg \
        .formal/tla/QGSDAccountManager.tla
```

Verify: `git status --short | grep "^A"` shows all 11 files staged.

### Task 2 — Update formal-verify.yml to invoke the master runner

Add a final step to `.github/workflows/formal-verify.yml` after all the individual tool steps:

```yaml
      - name: Run full FV pipeline via master runner (run-formal-verify.cjs)
        env:
          PRISM_BIN: /tmp/prism/bin/prism
        run: node bin/run-formal-verify.cjs
        continue-on-error: true
```

Use `continue-on-error: true` since the master runner may fail on CI due to missing TLC/Alloy JARs in some environments (same guard pattern as the existing `verify-quorum-health` step). The step exercises the pipeline wiring — failures are visible in CI logs.

### Task 3 — Commit

```
git commit -m "feat(v0.14-01): commit untracked FV tools, formal artifacts, and wire CI master runner (INTG-01, INTG-02, INTG-03)"
```
