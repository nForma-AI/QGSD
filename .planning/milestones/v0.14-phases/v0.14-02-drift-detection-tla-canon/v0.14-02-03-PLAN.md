## Background: Guard Name Mapping

The XState machine uses camelCase guard names (`minQuorumMet`, `noInfiniteDeliberation`, `phaseMonotonicallyAdvances`) defined in `guards:` in `qgsd-workflow.machine.ts`.

TLA+ uses PascalCase predicates/invariants (`MinQuorumMet`, `DeliberationBounded`, `AllTransitionsValid`). These are not the same names — they are semantic translations.

The mapping between XState guard names and their TLA+ equivalents is captured in `.formal/tla/guards/qgsd-workflow.json`:

```json
{
  "guards": {
    "minQuorumMet":            "successCount * 2 >= N",
    "noInfiniteDeliberation":  "deliberationRounds < MaxDeliberation",
    "phaseMonotonicallyAdvances": "TRUE"
  }
}
```

Guard names also appear literally in:
- `.formal/tla/QGSDQuorum.tla` header comment (lines 11-12): `minQuorumMet` and `noInfiniteDeliberation` by exact name
- `.formal/tla/QGSDQuorum_xstate.tla` (generated): in action comments like `\* COLLECTING_VOTES -[VOTES_COLLECTED / minQuorumMet]-> DECIDED`

**Check strategy for TASK-GC-01:** Verify each XState guard name appears in at least one of: (a) `guards/qgsd-workflow.json` keys, (b) `QGSDQuorum.tla` source text (camelCase comment references). This is a forward-check: XState → TLA+ spec artifacts. The `guards/qgsd-workflow.json` file is the authoritative bridge document.

// Check 5: Guard drift enforcement
// Verify each XState guard name is documented in the TLA+ guard mapping file.
//
// Mapping context:
//   XState uses camelCase guard names (minQuorumMet, noInfiniteDeliberation, phaseMonotonicallyAdvances).
//   TLA+ uses PascalCase predicates (MinQuorumMet, DeliberationBounded) — different names, same semantics.
//   The bridge is .formal/tla/guards/qgsd-workflow.json, which maps XState guard names to TLA+ expressions.
//   If a guard is renamed in the XState machine, it must also be updated in the guards JSON and QGSDQuorum.tla.
//
// Check: every xstateGuardNames entry must appear as a key in guards/qgsd-workflow.json.
const guardsJsonPath = path.join(ROOT, 'formal', 'tla', 'guards', 'qgsd-workflow.json');
if (xstateGuardNames.length > 0) {
  if (!fs.existsSync(guardsJsonPath)) {
    fail('.formal/tla/guards/qgsd-workflow.json not found — cannot verify guard name sync');
  } else {
    let guardsJson = null;
    try {
      guardsJson = JSON.parse(fs.readFileSync(guardsJsonPath, 'utf8'));
    } catch (e) {
      fail('Could not parse .formal/tla/guards/qgsd-workflow.json: ' + e.message);
    }
    if (guardsJson) {
      const mappedGuardNames = Object.keys(guardsJson.guards || {});
      ok('XState guard names: ' + xstateGuardNames.join(', '));
      ok('Mapped guard names (guards/qgsd-workflow.json): ' + mappedGuardNames.join(', '));

      // Forward check: XState guard → guards JSON (drift detection)
      const missingFromMapping = xstateGuardNames.filter(g => !mappedGuardNames.includes(g));
      if (missingFromMapping.length > 0) {
        fail(
          'XState guards not found in .formal/tla/guards/qgsd-workflow.json: ' + missingFromMapping.join(', ') +
          '\n         (Update guards/qgsd-workflow.json to map these guard names to their TLA+ predicates)'
        );
      }

      // Reverse check: guards JSON → XState (orphan detection)
      const orphanedGuards = mappedGuardNames.filter(g => xstateGuardNames.length > 0 && !xstateGuardNames.includes(g));
      if (orphanedGuards.length > 0) {
        fail(
          '.formal/tla/guards/qgsd-workflow.json references guards not in XState machine: ' + orphanedGuards.join(', ') +
          '\n         (These guard mappings are orphaned — remove them or restore the XState guard)'
        );
      }

      if (missingFromMapping.length === 0 && orphanedGuards.length === 0) {
        ok('Guard names match exactly between XState machine and guards/qgsd-workflow.json');
      }

      // Corroboration: check TLA+ source mentions at least minQuorumMet by name (comment reference)
      const tlaGuardRefs = xstateGuardNames.filter(g => tlaSrc.includes(g));
      if (tlaGuardRefs.length > 0) {
        ok('TLA+ source references guard names: ' + tlaGuardRefs.join(', '));
      }
    }
  }
} else {
  warn('No guard names extracted from XState machine — skipping guard drift check');
}
```

This check satisfies DRFT-02 ("catches transition and guard names, not just state list") and extends DRFT-03 to cover guard orphan detection bidirectionally.

// Check 4: Alloy orphan detection
// Scan Alloy .als files for:
//   (a) ALL_CAPS quoted string literals → cross-reference against xstateStateNames
//   (b) camelCase guard name occurrences (as identifiers or comment refs) → cross-reference against xstateGuardNames
const alloyDir = path.join(ROOT, 'formal', 'alloy');
if (fs.existsSync(alloyDir)) {
  const alsFiles = fs.readdirSync(alloyDir).filter(f => f.endsWith('.als'));
  for (const alsFile of alsFiles) {
    const alsSrc = fs.readFileSync(path.join(alloyDir, alsFile), 'utf8');

    // (a) State orphan scan: quoted ALL_CAPS string literals
    const alloyStateRefs = (alsSrc.match(/"([A-Z_]{3,})"/g) || [])
      .map(s => s.replace(/"/g, ''));
    const orphanedStates = alloyStateRefs.filter(s => xstateStateNames.length > 0 && !xstateStateNames.includes(s));
    if (orphanedStates.length > 0) {
      warn('Alloy ' + alsFile + ' references states not in XState machine: ' + [...new Set(orphanedStates)].join(', '));
    }

    // (b) Guard orphan scan: camelCase guard name occurrences
    // Scan for word-boundary occurrences of each mapped guard name in the Alloy source.
    // Uses warn() (not fail()) because Alloy models may abstract guard logic under different names.
    if (xstateGuardNames.length > 0) {
      const orphanedGuardRefs = xstateGuardNames.filter(g => {
        // Check if this guard name appears in the Alloy source (as a comment or identifier)
        // We only flag guard names that appear verbatim — PascalCase TLA+ predicates are not flagged here.
        const re = new RegExp('\\b' + g + '\\b');
        return re.test(alsSrc);
      });
      // Note: these are not orphans — they're confirmed references.
      // Orphans are guard names in Alloy that DON'T match any XState guard.
      // Since Alloy uses different naming (e.g., MajorityReached instead of minQuorumMet),
      // we look for verbatim camelCase guard name matches only.
      // This is best-effort: flag Alloy files that reference XState guard names (guard they map correctly).
      if (orphanedGuardRefs.length > 0) {
        ok('Alloy ' + alsFile + ' references XState guard names (verbatim): ' + orphanedGuardRefs.join(', '));
      }
    }
  }
  ok('Alloy orphan scan complete');
}
```

**Note on design decision (document in code comment):** Alloy guard orphan detection uses `warn()` not `fail()` because Alloy models are intentional abstractions — they may use different predicate names (e.g., `MajorityReached` instead of `minQuorumMet`). Guard name drift in Alloy is advisory only. The authoritative guard mapping check is in Check 5 (guards/qgsd-workflow.json).
