### Gaps Summary

The phase delivered most of its goal: the `_xstate` suffix fix (BROKEN-01) works correctly and is verified, CI hardening is in place, and the esbuild+require() extraction is implemented. State-level and orphan detection works bidirectionally.

The gap is narrow but material: **guard names and event/transition names are extracted from the XState machine via the esbuild walk, but no check verifies they appear in the formal specs (or vice versa)**. The variables `xstateGuardNames` and `xstateEventNames` are dead code — they are populated but never read by any check. This means renaming a guard in the XState machine (e.g., `minQuorumMet` → `quorumSufficient`) would not cause `npm test` to fail, contradicting SC-1, SC-2, and the DRFT-02 requirement text ("catches transition and guard names, not just state list").

The second test in `check-spec-sync.test.cjs` ("exits 1 when a state in the XState machine is missing from TLA+ TypeOK") is a no-op placeholder that passes trivially at 0ms without any assertion, which masks the absence of an actual drift-injection test.

These two gaps are related and have the same root cause: the implementation extracted the data correctly but did not add the corresponding check logic for guard names and transitions.
