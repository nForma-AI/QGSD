**Test 4 — Timing assertion (PERF-02):**

```js
test('completes within 120 seconds in no-Java environment (--only=generate filter)', () => {
  // In a no-Java environment, generate steps will fail (no Java for TLC) but will complete fast.
  // This test verifies the runner does not hang or exceed the expected wall-clock bound.
  // Full end-to-end timing with Java requires real tool execution (CI integration test).
  // Here we verify the runner exits within 120s on the generate subset as a proxy check.
  const startMs = Date.now();
  const result = spawnSync(process.execPath, [RUN_FV, '--only=generate'], {
    encoding: 'utf8',
    timeout: 120000,  // 120s hard cap — runner must not hang
  });
  const elapsedMs = Date.now() - startMs;

  // Should not time out (result.signal !== 'SIGTERM' and no timeout error)
  assert.strictEqual(result.signal, null, 'runner must not be killed by timeout');
  assert.ok(elapsedMs < 120000, 'runner completed in ' + elapsedMs + 'ms (limit: 120000ms)');

  // The output must contain the wall-clock line added in Task 1
  const output = (result.stdout || '') + (result.stderr || '');
  assert.match(output, /Wall-clock/i, 'output must include Wall-clock timing line');
});
```

**Test 5 — Integration smoke: all step IDs present in output (TASK-03 coverage):**

```js
test('integration smoke: all expected step IDs appear in output when --only=tla filter used', () => {
  // Verify that parallelization does not silently drop steps.
  // With --only=tla, the 8 TLA+ step IDs must all appear in stdout.
  // Steps will fail (no Java) but must be ATTEMPTED (id printed before execution).
  const TLA_STEP_IDS = [
    'tla:quorum-safety',
    'tla:quorum-liveness',
    'tla:oscillation',
    'tla:convergence',
    'tla:breaker',
    'tla:deliberation',
    'tla:prefilter',
    'tla:account-manager',
  ];

  const result = spawnSync(process.execPath, [RUN_FV, '--only=tla'], {
    encoding: 'utf8',
    timeout: 120000,
  });

  const output = (result.stdout || '') + (result.stderr || '');

  for (const stepId of TLA_STEP_IDS) {
    assert.ok(
      output.includes(stepId),
      'Expected step ID "' + stepId + '" to appear in output — step may have been silently dropped'
    );
  }
});
```

**Why --only=tla for smoke test (not full run):** Full run requires Java (TLC) and Alloy/PRISM binaries. The `--only=tla` filter exercises the parallelization code path (Phase 2 only, generate steps empty → skipped) without needing external tooling. The step IDs are printed at the START of each step's header before execution, so they appear even when Java is absent.
