Create bin/mismatch-register.cjs following the existing bin/ CJS script pattern.

The script must:
1. Read conformance-events.jsonl using planning-paths.cjs for path resolution
2. Read .planning/formal/evidence/event-vocabulary.json for vocabulary validation
3. Import mapToXStateEvent from bin/validate-traces.cjs (already exported in v0.29-01-03)
4. Import createActor and nfWorkflowMachine from dist/machines/nf-workflow.machine.js
5. For each conformance event:
   a. Map to XState event via mapToXStateEvent
   b. If unmappable (returns null), skip -- this is an instrumentation gap, not a mismatch
   c. Create a fresh XState actor, send the event, compare expected vs actual state
   d. Apply H1 methodology: mid-session events (event.phase !== 'IDLE' AND action !== 'quorum_start') are methodology_skip, NOT mismatches.
      // WHY fresh-actor-per-event: Provides isolation guarantees -- no cross-event state pollution.
      // Each event is evaluated independently against the model starting from IDLE.
      // WHY H1 methodology skip is the correct compensating mechanism: Fresh actors always
      // start in IDLE, so mid-session events (which expect non-IDLE starting states) would
      // produce false mismatches. H1 skip acknowledges this is a methodology limitation,
      // not a model disagreement, and is the correct way to handle it without introducing
      // cross-event state coupling.
   e. For genuine state disagreements, create a mismatch entry
6. Also incorporate existing .divergences.json entries (10 known divergences) -- classify each as "explained" if they match the H1 methodology skip pattern, "open" otherwise

**JSONL format (one JSON object per line, file must NOT start with '['):**
```
{"id":"MISMATCH-001","timestamp":"...","l2_source":"nf-workflow.machine.ts","l1_trace_ref":{"event_index":5,"session":"session-001"},"expected_state":"DECIDED","actual_state":"IDLE","divergence_type":"state_mismatch","resolution":"open","resolution_notes":null,"classification":null}
```

Resolution values: "open" | "explained" (known methodology limitation) | "bug" (genuine implementation bug)

**Output file:** .planning/formal/semantics/mismatch-register.jsonl (JSONL, NOT JSON array)
Also write a summary to stdout (or --json for machine-readable):
```json
{ "total_mismatches": N, "by_resolution": { "open": N, "explained": N, "bug": N }, "by_divergence_type": {...} }
```

**CRITICAL:** Use fs.writeFileSync for initial generation (overwrites). Future phases can use fs.appendFileSync for incremental updates. The JSONL format enables append-only updates without rewriting the whole file.

Create bin/observed-fsm.cjs following the existing bin/ CJS script pattern.

The script must:
1. Read conformance-events.jsonl using planning-paths.cjs
2. Read .planning/formal/evidence/event-vocabulary.json
3. Import mapToXStateEvent from bin/validate-traces.cjs
4. Import createActor and nfWorkflowMachine from dist/machines/nf-workflow.machine.js
5. Derive an observed FSM by replaying all mappable events using DUAL-MODE replay:

   **Mode A -- Per-event isolation (primary):**
   a. For each event, create a fresh XState actor, record beforeState, send mapped event, record afterState
   b. Build adjacency map: { fromState: { eventType: { toState, count } } }
   c. This captures IDLE->X transitions with full isolation (no cross-event state pollution)

   **Mode B -- Per-session running actor (supplementary):**
   a. Group conformance events by session_id
   b. For each session, create ONE actor and replay all events in sequence
   c. Record the full transition chain: state_0 -event_1-> state_1 -event_2-> state_2 ...
   d. Build a supplementary adjacency map capturing multi-step transition chains
   e. This captures non-IDLE starting states (e.g., quorum.collecting -> quorum.decided)

   **Merge:** The final observed_transitions combines both modes. Per-event mode is authoritative for counts; per-session mode adds transitions not visible from IDLE starts. Mark each transition with its source: "per_event", "per_session", or "both".

6. Track total events processed, events skipped (unmappable), and events per state
7. Extract the hand-written model's states and transitions from nfWorkflowMachine definition for comparison
8. Produce a structural diff:
   - missing_in_observed: transitions in the hand-written model that no trace exercises
   - missing_in_model: transitions observed in traces but not defined in the model
   - matching: transitions that both agree on
9. Track TWO coverage metrics (per research Pitfall 4):
   - model_coverage: what % of hand-written model transitions are exercised by traces
   - vocabulary_coverage: what % of conformance events map to known vocabulary actions

**Output format (observed-fsm.json) -- JSON data structure, NOT an XState machine definition:**
```json
{
  "schema_version": "1",
  "generated": "ISO timestamp",
  "observed_transitions": { "IDLE": { "QUORUM_START": { "to_state": "quorum.collecting", "count": N, "source": "both" } } },
  "states_observed": ["IDLE", "quorum.collecting", ...],
  "model_comparison": {
    "missing_in_observed": [...],
    "missing_in_model": [...],
    "matching": [...]
  },
  "coverage": {
    "model_coverage": 0.XX,
    "vocabulary_coverage": 0.XX,
    "total_events": N,
    "mapped_events": N,
    "unmapped_events": N
  },
  "replay_modes": {
    "per_event_transitions": N,
    "per_session_transitions": N,
    "sessions_replayed": N
  }
}
```
