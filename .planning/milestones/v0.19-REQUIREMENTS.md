# Requirements: QGSD v0.19 FV Pipeline Hardening

**Defined:** 2026-02-27
**Core Value:** Planning decisions are multi-model verified by structural enforcement, not instruction-following — a Stop hook that reads the transcript makes it impossible for Claude to skip quorum.

## v0.19 Requirements (All Satisfied)

### Unified Verdict Format (UNIF)

- [x] **UNIF-01**: All FV checkers append normalized JSON to `check-results.ndjson` — `{ tool, check_id, result, detail, ts }`
- [x] **UNIF-02**: `run-formal-verify.cjs` generates `check-results.ndjson` as canonical output artifact
- [x] **UNIF-03**: Triage bundle reads from `check-results.ndjson`, not tool stdout; CI enforcement steps run inside orchestrator before summary read (v0.19-11 timing fix)
- [x] **UNIF-04**: CI step exits non-zero when any `result=fail` entry exists in `check-results.ndjson`

### Calibration Governance (CALIB)

- [x] **CALIB-01**: `formal/policy.yaml` is the single authoritative source for PRISM calibration parameters
- [x] **CALIB-02**: `run-prism.cjs` reads `policy.yaml` and writes `writeCheckResult` with `observation_window` metadata
- [x] **CALIB-03**: `read-policy.cjs` exposes all policy fields with typed interface
- [x] **CALIB-04**: `policy.conservative_priors.tp_rate` and `policy.conservative_priors.unavail` from `formal/policy.yaml` wire directly to `run-prism.cjs` fallback constants (v0.19-10 fix)

### Liveness Fairness (LIVE)

- [x] **LIVE-01**: Liveness properties are detected from TLA+ spec `.cfg` files via `detectLivenessProperties()`
- [x] **LIVE-02**: `detectLivenessProperties` wired to all 5 TLC runners: `run-tlc.cjs`, `run-oscillation-tlc.cjs`, `run-breaker-tlc.cjs`, `run-protocol-tlc.cjs`, `run-account-manager-tlc.cjs`

### Enforcement Layer (REDACT / EVID / DRIFT)

- [x] **REDACT-01**: `check-trace-redaction.cjs` checks traces for PII/secret patterns and emits `check-results.ndjson`
- [x] **REDACT-02**: Redaction check runs as CI step in `run-formal-verify.cjs`
- [x] **REDACT-03**: Redaction violations emit `result=fail` (not warn)
- [x] **EVID-01**: `validate-traces.cjs` evidence output appends to `check-results.ndjson`
- [x] **EVID-02**: Evidence checks include `observation_window` metadata (window_start, window_end, n_traces)
- [x] **DRIFT-01**: `check-trace-schema-drift.cjs` detects schema drift and emits `check-results.ndjson`
- [x] **DRIFT-02**: Schema drift check runs as CI step in `run-formal-verify.cjs`

### MCP Environment Model (MCPENV)

- [x] **MCPENV-01**: `formal/spec/mcp-calls/` contains TLA+ model of MCP environment behavior
- [x] **MCPENV-02**: `MCMCPEnv` added to `SURFACE_MAP` and `VALID_CONFIGS`; `invariants.md` created with EventualDecision fairness declarations; CI step added to `formal-verify.yml`
- [x] **MCPENV-03**: MCP availability rates from `run-prism.cjs` feed into the MCP environment model
- [x] **MCPENV-04**: `readMCPAvailabilityRates` exported with composite-key filter; `module.exports` always reachable via require.main guard; prism:mcp-availability STEPS entry and CI step added

### R3.6 Improvement Iteration (IMPR)

- [x] **IMPR-01**: R3.6 quorum enforcement reviewed and improvement areas identified
- [x] **IMPR-02**: Quorum slot dispatch updated to parallel sibling Task calls (R3.2 requirement)
- [x] **IMPR-03**: Slot worker protocol hardened against missing/malformed responses
- [x] **IMPR-04**: Improvement validated with quorum test runs
