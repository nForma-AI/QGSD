## Decisions Made
- Each write site uses a pid-scoped tmp name (`absPath + '.' + process.pid + '.tmp'`) so concurrent processes cannot collide on the same tmp file
- merge-wave filters vote files by both round number and task name substring to prevent cross-task vote leakage when a shared `--dir` is used across multiple tasks
- merge-wave calls both `recomputeStats()` and `recomputeSlots()` after applying all votes in memory, so both slot-keyed and model-keyed stats are consistent after the merge
- merge-wave gracefully exits 0 with an informative message when the directory is nonexistent or contains no matching vote files, enabling safe use in orchestrator pipelines where waves may be empty
