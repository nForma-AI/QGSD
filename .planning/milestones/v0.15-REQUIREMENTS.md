# Requirements: QGSD v0.15 Health & Tooling Modernization

**Defined:** 2026-02-27
**Core Value:** Fix the GSD health checker to recognize QGSD's versioned phase naming convention, guard `--repair` against rich STATE.md data loss, archive legacy pre-versioning phase dirs, and surface quorum failure patterns in the health report.

## v0.15 Requirements

### Health Checker (HLTH)

- [x] **HLTH-01**: `validate health` produces zero W005 false positives for versioned phase directories (`v0.X-YY-name` format)
- [x] **HLTH-02**: `validate health` produces zero W007 false positives for versioned phase directories
- [x] **HLTH-03**: `validate health` produces zero W002 false positives for versioned phase references in STATE.md

### Repair Safety (SAFE)

- [x] **SAFE-01**: `--repair` cannot silently overwrite a rich STATE.md (>50 lines) without explicit `--force` flag; guard fires and repair is skipped with a message
- [x] **SAFE-02**: Legacy pre-versioning numeric phase dirs 18-39 are archived to `.planning/archive/legacy/`

### Visibility (VIS)

- [x] **VIS-01**: `/qgsd:health` surfaces W008 warning when a quorum slot has ≥3 failures recorded in `quorum-failures.json`
