**Alert fetching (GET /api/v1/provisioning/alert-rules):**
- Successful response with 3 alert rules (one alerting, one pending, one ok) → returns 3 issues:
  - id: "grafana-alert-{rule.id or index}"
  - title: alert rule name (rule.title)
  - severity: "error" for alerting state, "warning" for pending, "info" for ok/paused/no_data
  - url: endpoint + dashboard URL (dashboardUid-based) if available, else endpoint
  - age: formatAge from updated timestamp
  - created_at: rule.updated or current timestamp
  - meta: includes annotations summary, folder name, evaluation group
  - source_type: "grafana"
  - issue_type: "drift" (from config default)
- Auth token: when auth_env set and env var exists → Authorization: Bearer header sent
- Auth failure (401/403 response) → status: "error", issues: []
- Timeout → status: "error", issues: []
- Malformed JSON → status: "error", issues: []
- Empty rules array → status: "ok", issues: []

**State mapping edge cases:**
- Alert with state "alerting" → severity: "error"
- Alert with state "pending" → severity: "warning"
- Alert with state "normal" or "ok" → severity: "info"
- Alert with state "nodata" → severity: "warning"
- Alert with unknown state → severity: "info"

**Annotations and metadata:**
- Alert with annotations (summary, description) → meta includes annotation text
- Alert without annotations → meta is empty string or minimal context

```js
// Grafana source handler for /qgsd:observe
// Fetches alert rules from Grafana unified alerting API

async function handleGrafana(sourceConfig, options) {
  // 1. Extract config: endpoint, auth_env
  // 2. Resolve auth token from process.env[auth_env]
  // 3. Build headers with Bearer auth
  // 4. GET ${endpoint}/api/v1/provisioning/alert-rules
  // 5. Parse response JSON (array of alert rule objects)
  // 6. Map each rule to standard issue schema:
  //    - state → severity mapping
  //    - annotations → meta
  //    - dashboardUid → url construction
  // 7. Return standard schema
  // On error: fail-open return
}
```

Key implementation details:
- Use `options.fetchFn || globalThis.fetch`
- Grafana alert rule API returns array directly (not wrapped in {data: ...})
- State field location varies: rule.labels?.grafana_state or infer from rule data
- For unified alerting (Grafana 9+): each rule has title, uid, folderUID, updated, annotations, labels
- State may need to be fetched separately from /api/prometheus/grafana/api/v1/alerts for current firing state. Keep it simple: use the provisioning endpoint for rule definitions. If a `state_endpoint` config is provided, also fetch /api/prometheus/grafana/api/v1/alerts for live state.
- Dashboard URL: construct `${endpoint}/d/${dashboardUid}` if dashboardUid exists in rule
- Severity mapping: { alerting: "error", firing: "error", pending: "warning", normal: "info", ok: "info", nodata: "warning", paused: "info" }

CommonJS module.exports: { handleGrafana }
