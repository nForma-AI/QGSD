```js
// Prometheus source handler for /qgsd:observe
// Supports: /api/v1/alerts (active alerts) and /api/v1/query (PromQL)

async function handlePrometheus(sourceConfig, options) {
  // 1. Extract config: endpoint, auth_env, query, threshold, timeout
  // 2. Resolve auth token from process.env[auth_env] if set
  // 3. Build headers: { Authorization: `Bearer ${token}` } if token exists
  // 4. Choose mode: query field present → PromQL mode, else → alerts mode
  // 5. Fetch with fetchFn (options.fetchFn || global fetch)
  // 6. Parse response JSON
  // 7. Map to standard issue schema
  // 8. Return { source_label, source_type: 'prometheus', status: 'ok', issues }
  // On any error: return { ..., status: 'error', error: message, issues: [] }
}
```

Key implementation details:
- Use `options.fetchFn || globalThis.fetch` for testability
- Alerts mode: GET `${endpoint}/api/v1/alerts`, parse data.alerts array
- Query mode: GET `${endpoint}/api/v1/query?query=${encodeURIComponent(query)}`, parse data.result array
- Severity mapping from Prometheus labels: look for `severity` label, default to "warning"
- Include helper: formatAge from observe-handlers.cjs (import it, or inline a copy to avoid coupling)
- All results get issue_type from sourceConfig.issue_type (config loader defaults to "drift" for prometheus)
- Fail-open: try/catch entire function, return error schema on any failure

CommonJS module.exports: { handlePrometheus }
