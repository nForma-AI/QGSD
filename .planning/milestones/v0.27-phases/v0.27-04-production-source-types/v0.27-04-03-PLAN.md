**Query execution (POST /{index}/_search):**
- Config with endpoint + index (e.g., "logstash-*") → POST to ${endpoint}/${index}/_search
- Successful response with 3 hits → returns 3 issues:
  - id: "es-{hit._id or index}"
  - title: hit._source.message (truncated to 120 chars if longer)
  - severity: from hit._source.level (error→error, warn/warning→warning, info→info, debug→info)
  - url: endpoint + "/_cat/indices" (or empty — no natural URL for log entries)
  - age: formatAge(hit._source["@timestamp"])
  - created_at: hit._source["@timestamp"]
  - meta: JSON.stringify of hit._source excluding message and @timestamp (compact, max 200 chars)
  - source_type: "logstash"
  - issue_type: "issue" (default from config, per CONTEXT.md)

**Query construction:**
- Default query: filter by @timestamp >= now-1h and level in [error, warn]
- Config with custom filter field (e.g., filter.since: "24h") → adjusts time range
- Config with custom filter.levels: ["error"] → filters to only error level

**Auth and error handling:**
- auth_env set → sends Authorization header (support both "Bearer" and "ApiKey" via auth_type config, default "ApiKey")
- Auth failure (401/403) → status: "error", issues: []
- Timeout → status: "error", issues: []
- Malformed JSON → status: "error", issues: []
- Empty hits → status: "ok", issues: []
- Missing message field in hit → uses "No message" as title
- Missing level field → defaults to "info" severity

```js
// Logstash/Elasticsearch source handler for /qgsd:observe
// Queries Elasticsearch indices for log entries matching severity filters

async function handleLogstash(sourceConfig, options) {
  // 1. Extract: endpoint, index (default "logstash-*"), auth_env, auth_type, filter
  // 2. Resolve auth from env
  // 3. Build Elasticsearch DSL query body:
  //    { query: { bool: { filter: [
  //      { range: { "@timestamp": { gte: "now-1h" } } },
  //      { terms: { level: ["error", "warn"] } }
  //    ] } }, size: 50, sort: [{ "@timestamp": "desc" }] }
  // 4. POST ${endpoint}/${index}/_search with JSON body
  // 5. Parse response.hits.hits array
  // 6. Map each hit to standard issue schema
  // 7. Return standard schema
  // On error: fail-open return
}
```

Key implementation details:
- Use `options.fetchFn || globalThis.fetch`
- POST request with Content-Type: application/json
- Elasticsearch response shape: { hits: { total: { value: N }, hits: [ { _id, _source: { message, level, @timestamp, ... } } ] } }
- Time range from filter.since config (parse duration like "1h", "24h", "7d" — use Elasticsearch relative syntax "now-{since}")
- Level filter from filter.levels config (default: ["error", "warn"])
- size from filter.limit config (default: 50)
- Auth header format: "ApiKey {token}" or "Bearer {token}" based on auth_type config

CommonJS module.exports: { handleLogstash }
