## What Was Built

Four modules implementing the P->F residual sweep and supporting helpers:

1. **bin/compareDrift.cjs** — Compares production measurement (entry.meta.measured_value) against formal expected value. Fail-open: returns false when either value is null.

2. **bin/extractFormalExpected.cjs** — Parses formal_ref strings ("spec:module/file:param" or "requirement:ID") and loads parameter values from .cfg and .json spec files. Includes shared parseFormalRef() utility.

3. **bin/isNumericThreshold.cjs** — Determines if a formal_ref points to a numeric parameter (auto-updatable) vs a correctness invariant (requires investigation). Uses extractFormalExpected internally.

4. **bin/sweepPtoF.cjs** — P->F residual sweep function. Reads debt ledger, filters for acknowledged entries, compares against formal thresholds, returns residual count and detail object.
