# Phase v0.27-01 Plan 03: Ledger I/O and Retention Policy Summary

**Atomic read/write operations for debt ledger with fail-open behavior, retention policy for automatic archival of old resolved entries, and comprehensive integration test proving end-to-end lifecycle.**

## Overview

Implemented the persistence layer for QGSD's debt ledger:

1. **Ledger I/O Module (debt-ledger.cjs):** Atomic read/write with fail-open behavior prevents data loss on corruption
2. **Retention Policy Module (debt-retention.cjs):** Archives resolved entries older than 90 days (configurable) to JSONL
3. **Integration Test (qgsd-debt.test.cjs):** Validates complete lifecycle from creation through archival across all v0.27-01 modules

### Task 1: Ledger I/O and Retention Policy (TDD)

**RED Phase:** Created comprehensive test suite in `bin/debt-ledger.test.cjs` with 19 tests covering:

**Ledger I/O Tests (8 tests):**
- Reading nonexistent file returns empty ledger with schema_version "1" (fail-open)
- Reading corrupt JSON file returns empty ledger without crashing (fail-open)
- Write creates parent directory if needed (recursive mkdir)
- Write is atomic: uses temp file (.tmp) + rename, not direct write
- Reading back a written ledger preserves structure and content
- Schema version field is preserved across read/write cycles
- last_updated is set on every write
- Empty debt_entries array writes correctly

**Retention Policy Tests (11 tests):**
- Entry with status=open is NEVER archived regardless of age (tested with 365-day-old entry)
- Entry with status=acknowledged is NEVER archived regardless of age
- Entry with status=resolving is NEVER archived regardless of age
- Entry with status=resolved AND resolved_at older than cutoff IS archived
- Entry with status=resolved AND resolved_at newer than cutoff stays active
- Default max_age is 90 days
- Custom max_age parameter is honored (tested with 30 days)
- Archived entries are written in JSONL format (one JSON object per line, no outer array)
- JSONL append does not overwrite existing archive content
- Entry without resolved_at falls back to last_seen for cutoff calculation (per research)
- Empty ledger returns empty active and empty archived arrays

**GREEN Phase:** Implemented two modules:

**bin/debt-ledger.cjs exports:**
- `readDebtLedger(ledgerPath)`: Reads JSON file, returns empty ledger on missing/corrupt (fail-open). Logs errors to stderr.
- `writeDebtLedger(ledgerPath, ledger)`: Writes atomically (temp+rename), creates parent dirs, updates last_updated timestamp

**bin/debt-retention.cjs exports:**
- `applyRetentionPolicy(ledger, maxAgeDays = 90)`: Splits debt_entries into active/archived. Only archives resolved entries older than cutoff. Returns `{ active, archived }`.
- `writeArchive(archivedEntries, archivePath)`: Appends entries to JSONL file (one entry per line). Creates parent dir if needed.

## Key Implementation Decisions

1. **Fail-open read behavior:** Missing or corrupt files return empty ledger with current timestamp, preventing crashes and allowing graceful degradation
2. **Atomic write with temp+rename:** Prevents file corruption if process crashes during write
3. **Retention cutoff logic:** Uses resolved_at if present, falls back to last_seen per research spec
4. **JSONL archive format:** One entry per line (not JSON array) allows append operations without parsing entire file
5. **Status-based filtering:** Open/acknowledged/resolving entries NEVER archived regardless of age, only terminal (resolved) state considered for archival
6. **Default 90-day retention:** Balances historical data with unbounded growth prevention (identified as critical pitfall in Phase research)
