## Key Decisions

- GitHub and bash handlers use `execFileSync` (not exec) to prevent shell injection
- Sentry/sentry-feedback handlers return `status: "pending_mcp"` with `_mcp_instruction` object — pure functions that can't call MCP tools directly
- Exported mapper functions (`mapSentryIssuesToSchema`, `mapSentryFeedbackToSchema`) let the observe command bridge MCP results to standard schema
- All handlers use dependency injection (`options.execFn`) for testability

## Handler Interface

All four handlers return the same schema:
```
{ source_label, source_type, status: "ok"|"error"|"pending_mcp", issues: [...], error?, _mcp_instruction? }
```
