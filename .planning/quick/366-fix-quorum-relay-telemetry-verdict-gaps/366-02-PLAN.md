<objective>
Close the verdict semantic gap: make truncation-derived FLAG verdicts distinguishable from genuine FLAG verdicts and exclude them from consensus.

Purpose: The TLA+ model's ConsensusDistinguishesTruncatedVerdicts invariant requires that truncation-derived default_flag verdicts are detectable in consensus. Currently parseVerdict returns plain "FLAG" for both genuine and truncation-derived verdicts, and nf-stop.js treats them identically. This fix makes truncation-derived FLAGs visible as FLAG_TRUNCATED and excludes them from vote counting.
Output: Updated dispatch verdict emission, updated consensus logic, install-synced hook.
</objective>

Change to:
```javascript
let verdict        = parseVerdict(output, mode);
// TRUNC-03: distinguish truncation-derived FLAG from genuine FLAG
if (verdict === 'FLAG' && parseVerdict.lastTruncationNote) {
  verdict = 'FLAG_TRUNCATED';

```

This is a 2-line addition. The `parseVerdict.lastTruncationNote` side-channel (line 811) already detects `[OUTPUT TRUNCATED` markers. The only change is: when the side-channel fires AND the verdict is FLAG (the default when no verdict line is found), upgrade to FLAG_TRUNCATED.

APPROVE and REJECT are never affected -- this only fires when `verdict === 'FLAG'` AND truncation was detected.
