## Summary

Fixed 41 Gate A grounding failures caused by empty `requirement_ids` in check-results.ndjson.

### Root Cause

The `bin/requirement-map.cjs` file lacked entries for 41 formal models generated by `close-formal-gaps`. When `run-alloy.cjs` ran these checks, it called `getRequirementIds(checkId)` which returned `[]` for all 41 check_ids. The traceability matrix builder then could not link check results to requirements, leaving `latest_result: null` for all 56 affected requirement IDs. Gate A reported "no passing traces" for all 41 models.

### Changes

1. **bin/requirement-map.cjs** — Added 41 new check_id -> requirement_ids entries:
   - 38 Alloy model entries (e.g., `alloy:config-zero-providers` -> `['CONF-10']`)
   - 3 TLA+ model entries with correct lowercase MC-prefixed check_ids
     (e.g., `tla:mccheckpointgate` -> `['QUORUM-06']`)

2. **check-results.ndjson** — Patched 38 existing entries with correct `requirement_ids`
   (35 from initial patch + 3 TLA+ entries)

3. **traceability-matrix.json** — Regenerated; all 56 previously-null requirement IDs
   now show `latest_result: "pass"` with correct `check_id` links

4. **per-model-gates.json** — Refreshed; Gate A pass count: 139 -> 178

5. **gate-a-grounding.json** — Updated; grounding_score: 0.772 -> 0.989, target_met: true
