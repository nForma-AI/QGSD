### Flow D Gap — Shallow Merge for circuit_breaker

When a project `qgsd.json` sets only one `circuit_breaker` sub-key (e.g., `{ oscillation_depth: 2 }`), the shallow merge at `config-loader.js:114` causes the project object to entirely replace the global `circuit_breaker` object. The other sub-key (`commit_window`) falls back to `DEFAULT_CONFIG` (value 6), not the global config value. This behavior is documented for `required_models` but not for `circuit_breaker`.

**Affects:** CONF-09

### Integration Gaps

**GAP-1 (Medium): Uninstall does not remove PreToolUse hook**

After `npx qgsd --uninstall`, `settings.json` retains the `PreToolUse` hook entry pointing to a deleted `qgsd-circuit-breaker.js` file. Claude Code will emit errors on every Bash command for users who uninstall.

Affects: INST-08

**GAP-2 (Low): `--reset-breaker` path mismatch**

The hook writes state to `gitRoot/.claude/circuit-breaker-state.json`. The CLI resolves from `process.cwd()/.claude/`. If the user runs `npx qgsd --reset-breaker` from a project subdirectory (not the root), the state file is not found and the command reports "no active state" without clearing the breaker.

Affects: RECV-01

**GAP-3 (Low): INST-10 sub-key backfill incomplete**

Reinstall checks `!existingConfig.circuit_breaker` (top-level key). If a user has `circuit_breaker: { oscillation_depth: 5 }` (missing `commit_window`), the block exists so the backfill is skipped. The missing sub-key is later defaulted by `validateConfig()` at runtime — functionally safe but the installer guarantee ("adds missing block without overwriting user-modified values") is only partially met for partial blocks.

Affects: INST-10
